Printing On Linux

Printing in Linux requires software to act as a translator, converting data from your applications into a language the printer can understand. The standard for this in Linux is the CUPS (Common UNIX Printing System).

On modern Linux desktops, printer installation is generally easy and automatic. When you connect a new printer, your system uses CUPS in the background to detect and configure it with minimal user effort. While the process is simple for the user, CUPS is the powerful system working behind the scenes to manage print jobs and communicate with the hardware.

CUPS Overview

What CUPS Does

CUPS (Common UNIX Printing System) is the standard, behind-the-scenes software that manages all printing on Linux. It acts as both a universal translator and a print server.

  • As a translator, it takes the document from any application (like a web browser or office suite) and translates it into the specific language your printer model understands.
  • As a print server, it manages the queue of print jobs for both local printers (plugged into your computer) and network printers.

Its modular design is what allows it to support a huge variety of printers from different manufacturers, each with its own unique print language.

The User Experience

The main benefit of CUPS is that it makes printing simple. For most users, the process is almost invisible.

You typically only need to configure a printer the first time you use it. In many cases, CUPS features automatic detection, meaning it will find and set up a new printer you connect to your computer or network with little to no effort on your part.

How Dos CUPS Work?

The CUPS printing system is made up of several key components that work together to process a print job, from the moment you click "print" until the page comes out of the printer.

  • Scheduler: This is the "brain" of CUPS. It's the central process that manages the print queue, processes incoming jobs, and directs the entire workflow.
  • Configuration files: These are the "rule-books" that store all the settings for your printers, printer classes, and the CUPS server itself.
  • Job files: These files contain the actual data to be printed, along with instructions on how to print it (e.g., number of copies, page size).
  • Log files: These files are the "diaries" of the printing system, recording the activity of the scheduler and any errors that occur. They are essential for troubleshooting.
  • Filter: A filter is a special program that translates the document you want to print (like a PDF or an image) into a final format that a specific printer model can understand. This is the core of the conversion process.
  • Printer drivers: These files provide CUPS with specific information about a printer's capabilities, such as the resolutions it supports and what data formats it can handle.
  • Backend: The backend is the component that sends the final, translated print data to the physical printer, whether it's connected via USB, over the network, or through another channel.

Scheduler

The CUPS Scheduler

The core component of CUPS is the scheduler. It acts as the "brain" or "air traffic controller" for your entire printing system. This central process is responsible for:

  • Managing the queue of print jobs.
  • Handling administrative commands.
  • Allowing users to check on printer status.
  • Controlling the flow of data through all the different parts of the printing process.

The Browser-Based Interface

To make management easy, CUPS provides a built-in browser-based interface. By navigating to a specific address in your web browser (usually http://localhost:631), you can access a control panel that allows you to view the print queue, check the status of your jobs, and manage pending print jobs.

Configuration Files

All CUPS configuration files are located in the /etc/cups/ directory. While there are several files, the two most important ones control the server's behavior and the settings for individual printers.

cupsd.conf: Server-Wide Settings

This is the main configuration file for the CUPS server itself. It contains all the system-wide settings, which are mostly related to networking and security, such as:

  • Which computers on the network can access the printers.
  • How printers are advertised on the network.
  • What management features are enabled.

This file does not contain any details about your specific printers.

printers.conf: Printer-Specific Settings

This file contains all the printer-specific details for every printer you have installed on your system. It describes each printer's status, capabilities, and configuration.

Important: This file is generated and managed automatically by CUPS whenever you add, remove, or modify a printer. You should not be modified by hand.

Job Files

When you submit a print job, CUPS creates temporary job files and places them in a queue located at /var/spool/cups/.

Each print job is represented by two separate files, distinguished by a letter prefix:

  • d (data) files: Contain the actual content of the document to be printed.
  • c (control) files: Contain the instructions for the print job, like the destination printer and the number of copies.

The collection of jobs waiting to be printed is known as the print queue. Each job in the queue is represented by temporary data files.

Once a job is sent to the printer and completes successfully, its data files are automatically removed from the queue.

Log Files

The CUPS printing system maintains log files to record all of its activities, which are essential for troubleshooting printing issues. These logs are stored in the /var/log/cups/ directory.

They contain detailed records, including access information, any error messages, and a log of every page that has been printed. Since they are plain text files, you can view them with standard command-line tools like less or tail.

user@lfs101: ~
$ sudo ls -l /var/log/cups

Filters, Printer Drivers, and Backends

This is the final stage of the CUPS printing process, where the print job is prepared and sent to the physical printer. It involves three key components.

The Final Processing Components

  • Filters: These are the "translators" of the printing system. A filter takes the data from your print job file and converts it into the specific language or format that your printer model understands.
  • Printer Drivers: These are description files (usually ending in .ppd) that act as the "instruction manual" for a specific printer. CUPS uses them to understand a printer's unique capabilities, such as its supported resolutions and paper sizes.
  • Backend: This is the "delivery truck." The backend is the software that takes the final, translated data and sends it to the physical printer over the correct connection (e.g., USB, network).

The Overall Workflow

Putting it all together, the entire printing process looks like this:

  1. When you print, the Scheduler creates a job file.
  1. The job is then sent to a Filter.
  1. The Filter uses the Printer Driver as a reference to correctly translate the job into the printer's language.
  1. The translated data is handed off to the Backend, which sends it to the printer.
  1. All of this activity is recorded in the log files.

Managing CUPS

The CUPS printing system runs as a background daemon (or service). You can control this service using the standard systemctl utility, which requires sudo for actions that change the service's state.

  • systemctl status cups: Checks the current status of the CUPS service (e.g., if it's running, enabled, or has failed).
  • sudo systemctl start cups: Starts the service in the current session.
  • sudo systemctl stop cups: Stops the service in the current session.
  • sudo systemctl restart cups: Restarts the service (a quick way to stop and then start it again, often used after changing a configuration).
  • sudo systemctl enable cups: Enables the service to start automatically every time the system boots.
  • sudo systemctl disable cups: Disables the service from starting automatically at boot.

Configuration a Printer Form The GUI

Adding and configuring a printer on a modern Linux desktop is typically a simple, graphical process.

The Simple Setup

Each Linux distribution provides a graphical configuration application for managing printers. In most cases, the process is straightforward:

  1. Ensure the printer is powered on and connected to your computer (e.g., via USB).
  1. Open your system's printer settings.
  1. The system should automatically detect the printer and guide you through a quick setup.

Troubleshooting Tips

If your printer does not appear automatically, the first step is to verify that the system can see the hardware. For a USB printer, you can open a terminal and run the lsusb command. If the printer is listed in the output, the system recognizes the connection.

Thanks to modern standardization (CUPS), you rarely need to install extra drivers or software from the printer's manufacturer, though this was more common in the past.

Adding Printers from the CUPS Web Interface

The CUPS printing system includes a powerful, built-in web interface that allows you to manage all aspects of your printing setup directly from a web browser.

How to Access It

You can access the web interface by navigating to the following address in your browser: http://localhost:631

What You Can Do

The web interface provides a full suite of tools for both printer and job management, including:

  • Printer Administration: You can add, remove, and configure both local and network printers. You can also set up a printer to be shared across your network.
  • Job Management: You can monitor the print queue in real-time, view a list of completed jobs, and cancel or reorder pending print jobs.

Authentication

For administrative actions, such as adding or modifying a printer, the web interface will prompt you for a username and password. On most Linux systems, you will need to authenticate using the root account and its password.

Printing Operations

Printing from the Graphical Interface

Printing from a graphical application in Linux is a straightforward, multi-step process.

  1. Open the Print Dialog: In most applications, you can open the print dialog using the keyboard shortcut CTRL+P.
  1. Configure Options: In the dialog window that appears, you first select your desired printer. You can then adjust various options such as page setup, print quality, and color. This is also where you would choose to "print to a file" (e.g., to create a PDF) instead of sending the job to a physical printer.
  1. Submit and Monitor: After clicking "Print," the job is submitted to the CUPS printing system. You can then monitor its status in the print queue by opening a web browser and navigating to the CUPS web interface at http://localhost:631.

Printing From Command-Line Interface

CUPS offers two traditional command-line interfaces for printing, inherited from the major families of the UNIX operating system.

The Two Commands

  • lp: The print command from the System V tradition of UNIX.
  • lpr: The print command from the BSD tradition of UNIX.

While both commands are available and can print various file types (like text, PDF, and images), lp is often just a front-end that passes its instructions to lpr. Their functionality is therefore very similar.

Primary Use Case: Automation

The main advantage of using command-line printing tools like lp and lpr is for automating tasks. They are essential when you need to print from within shell scripts or other automated workflows where a graphical interface is not available or practical.

Using lp

The command line provides a powerful way to manage printing, offering the same capabilities as the graphical interface through two main commands: lp and lpoptions.

lp: Sending a Print Job

The lp command is used to print a file from the command line. It accepts various options that allow you to specify details for the print job, such as the destination printer, the number of copies, and page settings—mirroring all the functions available in a graphical print dialog.

lpoptions: Configuring Defaults

The lpoptions command is a utility used to view and set default options for your printers. You can use it to configure:

  • System-wide defaults, such as setting which printer is the default printer for your entire system.
  • Printer-specific defaults, like the default paper size or number of copies for a particular printer.

To see a list of the options your printer supports, you can run the lpoptions help command.

CommandUsage
lp <filename>To print the file to default printer
lp -d printer <filename>To print to a specific printer (useful if multiple printers are available)
program | lpecho string | lpTo print the output of a program
lp -n number <filename>To print multiple copies
lpoptions -d printerTo set the default printer
lpq -aTo show the queue status
lpadminTo configure printer queues

Managing Print Jobs

If your print job is stuck in a long queue, Linux provides command-line tools for print job management that can help. These utilities allow you to interact with the print queue directly from your terminal to perform tasks such as:

  • Monitoring the status of all jobs.
  • Listing available printers and checking their status.
  • Canceling a pending print job.
  • Moving a job from one printer's queue to another.
CommandUsage
lpstat -p -dTo get a list of available printers, along with their status
lpstat -aTo check the status of all connected printers, including job numbers
cancel job-id or lprm job-idTo cancel a print job
lpmove job-id newprinterTo move a print job to new printer

Manipulating Postscript and PDF Files

Working With Postscript and PDF

What is PostScript?

PostScript is a powerful page description language developed by Adobe in the 1980s. Think of it as a set of text-based programming instructions that tell a printer exactly how to draw a page, including text, shapes, and images. Its main strength is its excellent handling of scalable fonts and vector graphics, which ensures high-quality printouts at any size or resolution.

Its Modern Role & Relationship to PDF

While once a dominant standard, PostScript has been largely superseded by its successor, PDF (Portable Document Format). The primary reason for this shift is that PDF files are compressed and therefore much smaller and more efficient for sharing and viewing.

However, PostScript is still highly relevant today. It is often used as a powerful intermediate format in printing and document conversion workflows. For example, an application might generate a PostScript file, which is then processed by a system like CUPS or converted into a final PDF document.

Working With enscript

The enscript command is a powerful tool used to convert plain text files into nicely formatted PostScript documents, ready for printing. It also supports other output formats like RTF and HTML.

It allows you to apply various formatting options during the conversion. For example, the following command converts a text file into a two-column, landscape PostScript file:

user@lfs101: ~
$ enscript -2 -r -p output.ps input.txt
  • 2: Formats the text into two columns.
  • r: Rotates the page to landscape mode.
  • p output.ps: Specifies the name for the output file.
CommandUsage
enscript -p psfile.ps textfile.txtConvert a text file to PostScript (saved to psfile.ps)
enscript -n -p psfile.ps textfile.txtConvert a text file to n columns where n=1-9 (saved in psfile.ps)
enscript textfile.txtPrint a text file directly to the default printer

Converting Between Postscript and PDF

While most users today work with PDF files, the PostScript format is still important, especially as an intermediate step in document processing. Linux provides several simple command-line utilities to convert between these two formats.

Standard Conversion Utilities

There are two main sets of tools designed specifically for this task. For most simple conversions, it doesn't matter which set you use as they are largely interchangeable.

  • From the widely-available ghostscript package, you have:
    • ps2pdf: Converts a PostScript file to PDF.
    • pdf2ps: Converts a PDF file to PostScript.
  • From the poppler package (which you may need to install), you have similar tools:
    • pstopdf: Converts PostScript to PDF.
    • pdftops: Converts PDF to PostScript.

The ImageMagick Alternative

In addition to the dedicated tools, the powerful convert command from the ImageMagick package (or gm convert on systems with Graphics Magick) can also be used to handle these and many other file format conversions.

CommandUsage
pdf2ps file.pdfConverts file.pdf to file.ps
ps2pdf file.psConverts file.ps to file.pdf
pstopdf input.ps output.pdfConverts input.ps to output.pdf
pdftops input.pdf output.psConverts input.pdf to output.ps
convert input.ps output.pdfConverts input.ps to output.pdf
convert input.pdf output.psConverts input.pdf to output.ps

Viewing PDF Content

Linux has excellent built-in support for both creating and reading PDF files. Applications like the LibreOffice suite can easily export documents as PDFs, and several high-quality open-source readers are available.

Common PDF Readers

The two most common PDF readers on Linux are:

  • evince: The most widely used and default PDF reader on many distributions, especially those using the GNOME desktop.
  • okular: The powerful and feature-rich default reader for the KDE desktop environment.

Both of these applications are robust and can also read other document formats, such as PostScript.

A Note on Adobe Acrobat Reader

The proprietary Adobe Acrobat Reader is no longer available for modern Linux systems. It was historically considered unstable and poorly maintained on the platform, and the native open-source readers like evince and okular are now the superior and recommended choices.

Manipulating PDFs

For advanced PDF operations that a standard viewer can't handle, Linux offers several powerful command-line utilities.

What You Can Do

These tools allow you to script and automate a wide range of tasks, including:

  • Merging multiple PDF files into one.
  • Splitting a single PDF into multiple files or pages.
  • Rotating pages.
  • Repairing corrupted PDF files.
  • Encrypting and decrypting documents.
  • Managing a PDF's metadata.

The Tools

There are three main programs available for these tasks:

  • qpdf: A modern, powerful, and feature-rich tool that is widely available on most Linux distributions.
  • pdftk-java: A popular tool that was once unavailable but has now been rewritten in Java and is available again on most systems. (It was formerly known as pdftk).
  • ghostscript (gs): Another very powerful and well-maintained tool, though its command-line syntax can be more complex than the others.

Using qpdf

You can accomplish a wide variety of tasks using qpdf including:

CommandUsage
qpdf --empty --pages 1.pdf 2.pdf -- 12.pdfMerge the two documents 1.pdf and 2.pdf. The output will be saved to 12.pdf.
qpdf --empty --pages 1.pdf 1-2 -- new.pdfWrite only pages 1 and 2 of 1.pdf. The output will be saved to new.pdf.
qpdf --rotate=+90:1 1.pdf 1r.pdfqpdf --rotate=+90:1-z 1.pdf 1r-all.pdfRotate page 1 of 1.pdf 90 degrees clockwise and save to 1r.pdf. Rotate all pages of 1.pdf 90 degrees clockwise and save to 1r-all.pdf
qpdf --encrypt mypw mypw 128 -- public.pdf private.pdfEncrypt with 128 bits public.pdf using as the passwd mypw with output as private.pdf.
qpdf --decrypt --password=mypw private.pdf file-decrypted.pdfDecrypt private.pdf with output as file-decrypted.pdf.

Using pdftk

As mentioned earlier, Marc Vinyals has developed and maintained a port to Java for pdftk which can be found on GitLab, together with instructions for installation. Some distributions, such as Ubuntu, may install this version only.

CommandUsage
pdftk 1.pdf 2.pdf cat output 12.pdfMerge the two documents 1.pdf and 2.pdf. The output will be saved to 12.pdf.
pdftk A=1.pdf cat A1-2 output new.pdfWrite only pages 1 and 2 of 1.pdf. The output will be saved to new.pdf.
pdftk A=1.pdf cat A1-endright output new.pdfRotate all pages of 1.pdf 90 degrees clockwise and save result in new.pdf.

Encrypting PDF Files With pdftk

ou can add a password to a PDF file to protect its confidential information using the pdftk (or pdftk-java) command. This is done with the user_pw PROMPT option, which tells the program to ask you for a password.

The basic command is:

user@lfs101: ~
$ pdftk input.pdf output output_protected.pdf user_pw PROMPT

When you run this command, it will prompt you enter and confirm a password. It then creates a new file that is an exact, password-protected copy of the original. The original file is not changed.

Using Ghostscript

What is Ghostscript?

Ghostscript (executed with the gs command) is a powerful, low-level interpreter for both PostScript and PDF files. It's an extremely versatile, all-purpose tool that can perform the same tasks as qpdf and pdftk (like merging and splitting files) and many other advanced manipulations.

The Trade-off: Power vs. Complexity

The main drawback of Ghostscript is its complexity. The command-line syntax is not intuitive, and the options required to perform even simple tasks can be very long and difficult to remember.

While it is an essential and powerful backend for many applications, for everyday command-line tasks like merging or splitting PDFs, tools like qpdf are often much easier to use.

Using Additional Tools

In addition to the major conversion and manipulation utilities, Linux offers several other specialized tools for handling PDF and related document formats.

pdfinfo

pdfinfo is a simple command-line utility used to extract information and metadata from a PDF file, such as the title, author, page count, and security settings. It's especially useful for quickly inspecting large files without opening them or when working in a command-line only environment.

flpsed

flpsed is a specialized tool designed to add data to PostScript documents. Its primary use case is for filling in forms or adding short annotations to a PostScript file before it's printed or converted to a PDF.

pdfmod

pdfmod is a simple graphical application for modifying PDF documents. It provides an intuitive interface for common tasks, allowing you to:

  • Reorder, rotate, and remove pages.
  • Export images from a document.
  • Edit metadata like the title, subject, and author.
  • Combine multiple documents using a simple drag-and-drop action.

For example, to collect the details of a document, you can use the following command:

user@lfs101: ~
$ pdfinfo /usr/share/doc/readme.pdf

Summary

Linux uses the Common UNIX Printing System (CUPS) as its standard printing service. It handles everything from managing printers to processing documents for printing.

The CUPS Printing System

CUPS provides several interfaces for managing printers and print jobs.

  • Command Line:
    • lp and lpr: The two primary commands used to send a file to a printer.
    • lpoptions: A utility for setting printer defaults, such as the default printer for your system.
  • Web Interface: CUPS includes a full-featured web-based administration tool, accessible at http://localhost:631.

Key Document Formats & Conversion

  • PostScript: A powerful page description language that excels at handling scalable fonts and vector graphics to produce high-quality prints.
  • PDF (Portable Document Format): The universal standard for exchanging documents in a way that preserves their appearance across different systems.
  • enscript: A command-line utility used to convert plain text files into nicely formatted PostScript documents.

PDF & PostScript Manipulation Toolkit

Linux offers a variety of powerful tools for editing and managing PDF and PostScript files.

  • pdftk: A versatile, all-in-one command-line tool for PDF manipulation. It can merge, split, encrypt, decrypt, and repair PDF files, as well as manage their metadata.
  • pdfinfo: A simple command-line utility that extracts information and metadata from a PDF file.
  • pdfmod: A user-friendly graphical application for modifying PDFs, allowing you to reorder, rotate, and delete pages.
  • flpsed: A specialized tool for adding text to PostScript documents, which is particularly useful for filling out forms.
