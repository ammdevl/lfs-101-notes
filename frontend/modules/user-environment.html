Accounts, Users and Accounts

Identifying the Current User

Linux is a multi-user operating system, meaning more than one user can log on at the same time.

  • To identify the current user, type whoami.
  • To list the currently logged-on users, type who.

Giving who the -a option will give more detailed information.

user@lfs101: ~
$ whoami
username
$ who
username  seat0        2025-10-09 12:34 (login screen)
username  tty2         2025-10-09 12:34 (tty2)

User Startup Files

When you start a command shell like bash, it runs startup files to configure your environment automatically.

There are two levels of these configuration files:

  • Global (/etc): These files contain settings that apply to all users on the system.
  • User (~/): Each user has their own startup files in their home directory. These can add to or override the global settings.

These files are used to personalize the shell, for tasks like customizing the command prompt, creating command aliases (shortcuts), and setting the default text editor.

Order of the Startup Files

Linux uses different startup files depending on whether you are in a login shell (when you first log in) or a non-login shell (every new terminal window you open).

  • Login Shell: Runs /etc/profile first, then the first one it finds of ~/.bash_profile, ~/.bash_login, or ~/.profile.
  • Non-Login Shell: Runs only ~/.bashrc each time a new terminal is opened.

Pro Tip: Because you want your customization (like aliases and prompts) to be available in every terminal, the best practice is to put all your personal settings into ~/.bashrc. Most systems are set up so the login files automatically run ~/.bashrc anyway. >_

An alias is a way to create a shortcut or a custom nickname for a command. You should add your aliases to your ~/.bashrc file to make them permanent. 🪄

Here are the basic commands:

  • unalias: Removes an alias.

Basics of Users and Groups

In Linux, every user is assigned a unique number called a user ID (uid).

To manage permissions efficiently, users are organized into groups. Each group also has a unique number, called a group ID (gid). Access to files and devices is controlled based on a user's uid and the gids of the groups they belong to.

This system is managed by two key files:

  • /etc/passwd: Links usernames to their uid and primary gid.
  • /etc/group: Defines each group, its gid, and lists its members.

Adding and Removing Users

While you can manage users through graphical tools, it's often done from the command line, especially in scripts. You'll need root privileges (sudo).

  • useradd: Adds a new user. For example, sudo useradd bjmoose creates the user, sets up their home directory (/home/bjmoose), and configures default settings.
  • userdel: Removes a user. By default, userdel bjmoose will remove the user but leave their home directory. To remove both the account and the home directory, you must use the r option: sudo userdel -r bjmoose.
  • id: Displays user information. Running id by itself shows your own uid and group memberships, while id username shows information for another user.
user@lfs101: ~
$ sudo ueradd username

Adding and Removing Groups

Adding a new group is done with groupadd:

user@lfs101: ~
$ sudo /usr/sbin/groupadd anewgroup

The group can be removed with:

user@lfs101: ~
$ sudo /usr/sbin/groupdel anewgroup

Adding a user to an already existing group is done with usermod. For example, you would first look at what groups the user already belongs to:

user@lfs101: ~
$ groups rjsquirrel
rjsquirrel : rjsquirrel

and then add the new group:

user@lfs101: ~
$ sudo /usr/sbin/usermod -a -G anewgroup rjsquirrel
$ groups rjsquirrel
rjsquirrel: rjsquirrel anewgroup

You can manage group memberships and properties from the command line.

  • usermod: Modifies a user's group memberships.
    • Adding a user to a group: When adding a user to a new group, you must use the a (append) option. If you forget a, the user will be removed from all other groups they were in!
    • Removing a user from a group: This is tricky. You must use the G option and provide a complete, comma-separated list of all the groups you want the user to remain a member of.
  • groupmod: Modifies the group itself. You can use it to change a group's name with the n option or its ID (gid) with the g option.
user@lfs101: ~
$ sudo /usr/sbin/usermod -G rjsquirrel rjsquirrel
$ groups rjsquirrel
rjsquirrel : rjsquirrel

only the rjsquirrel group will be left.

The root User

The root account, also known as the superuser or administrator, is the most powerful account in Linux with full access to everything on the system.

Because it's so powerful, giving a user full root access is extremely risky and rarely necessary. A safer and better approach is to use the sudo command, which can grant limited administrative privileges to a user:

  • Only on a temporary basis for a single command.
  • Only for a specific, limited set of commands.

su and sudo

There are two main commands for gaining administrative privileges, but one is much safer than the other.

  • su: This command switches you to another user's account, usually root. This gives you a full shell session with unlimited power, which is considered very dangerous because it's easy to accidentally damage the system.
  • sudo: This is the safer and preferred method. It grants root privileges for only a single command at a time, which greatly reduces the risk of making a mistake.

Elevating to root Account

The practical difference between su and sudo is how long you need elevated privileges.

  • Use su when you need to run a series of commands as the superuser. You'll be prompted for the root password and will remain in a root shell until you exit.
  • Use sudo to run only one command with root privileges. You'll immediately return to being a normal user after the command finishes.

The configuration for who can use sudo and what they can do is stored in the /etc/sudoers file and the /etc/sudoers.d directory.

Environment Variables

Overview

An environment variable is a named string of text that contains a value used by the command shell and other applications. Think of them as system-wide settings that programs can read.

Their values can either be preset by the system or set by the user, either temporarily at the command line or permanently in a startup script.

You can view all the currently set environment variables by using the commands set, env, or export. The set command will often display much more information than the other two.

Setting Environment Variables

By default, variables created in a shell script are private to that script. Any child processes (or sub-shells) started by the script won't have access to them.

To make a variable visible to child processes, you must use the export command. This command effectively promotes the variable from local to global for any processes spawned by that shell.

The HOME Variable

The HOME environment variable contains the path to your home directory.

The system uses this as a default location, and you can quickly navigate there in a few ways. Typing cd with no arguments, cd $HOME, and cd ~ are all equivalent commands that will take you directly to your home directory. The tilde (~) is a very common shortcut for the value of $HOME.

The PATH Variable

The PATH environment variable is a colon-separated list of directories that tells your shell where to look for programs.

When you type a command like ls, the shell searches each directory in the PATH in order until it finds the executable file. This is why you don't have to type the full path, like /usr/bin/ls, every time.

You can add your own directories to it. For example, to add a personal bin directory to the beginning of the search path, you would use the command: export PATH=~/bin:$PATH

The SHELL Variable

The SHELL environment variable stores the full path to your default command shell. 🐚 This is the program that processes the commands you type in a terminal. For most Linux users, its value is typically /bin/bash.

user@lfs101: ~
$ echo $SHELL
/bin/bash

The PS1 Variable and the Command Line Prompt

The PS1 environment variable controls what your command line prompt looks like. You can customize it by setting it to a string that includes special characters, which must be enclosed in single quotes.

Common special characters include:

  • \u for your username
  • \h for the hostname
  • \w for the current working directory

For example, to get a prompt that looks like user@host:directory$, you would use the command: export PS1='\u@\h:\w$ '

To temporarily change your prompt, it's a good practice to first save the old one (OLD_PS1=$PS1) so you can easily restore it later (PS1=$OLD_PS1).

Recalling Previous Commands

Overview

Your bash shell keeps a history of the commands you've recently used, which is saved in the ~/.bash_history file in your home directory.

You can easily recall previous commands by pressing the Up and Down arrow keys. To see the full numbered list of commands, simply type the history command.

Note that if you have multiple terminals open, the history from a specific session is only saved to the .bash_history file when that terminal is closed.

Using History Environment Variables

You can control the behavior of the bash command history by setting several environment variables.

  • HISTFILE: Sets the location of the history file (e.g., ~/.bash_history).
  • HISTFILESIZE and HISTSIZE: Determine the maximum number of commands to save in the file and to remember in the current session, respectively.
  • HISTCONTROL: Defines rules for saving commands, such as ignoring duplicates.
  • HISTIGNORE: Specifies patterns of commands that should never be saved to the history.

File Permissions

File Ownership

In Linux and other UNIX-based operating systems, every file is associated with a user who is the owner. Every file is also associated with a group (a subset of all users) which has an interest in the file and certain rights, or permissions: read, write, and execute.

CommandUsage
chownUsed to change user ownership of a file or directory
chgrpUsed to change group ownership
chmodUsed to change the permissions on the file, which can be done separately for owner, group and the rest of the world (often named as other)

File Permission Modes and chmod

In Linux, files have three types of permissions that determine who can access them and how.

The permissions are read (r), write (w), and execute (x). These rules apply to three classes of users: the user who owns the file (u), the group the file belongs to (g), and others (o).

You can change these permissions using the chmod command, which has two common methods.

Symbolic Method

This method uses letters and symbols to add (+) or remove (-) specific permissions for different users. While precise, it can be complex to type.

For example: chmod uo+x,g-w somefile gives execute permission to the user and others, and removes write permission from the group.

Numeric (Octal) Method

This is a common shorthand that sets all permissions at once using a three-digit number. Each digit represents the permissions for the user, group, and others, respectively. The digit is calculated by adding these values:

  • 4 for read
  • 2 for write
  • 1 for execute

So, a permission of 7 (4+2+1) is read, write, and execute (rwx), while 5 (4+1) is read and execute (r-x).

For example: chmod 755 somefile sets the permissions to:

  • User: 7 (rwx)
  • Group: 5 (r-x)
  • Others: 5 (r-x)

Summary

Users & Privileges

Linux is a multi-user system where every user has an account.

  • who: Lists all users currently logged on to the system.
  • whoami: Shows your own current user ID.

The root account (or superuser) has complete control over the system. Granting full root access to a regular user is extremely dangerous. The proper way to perform administrative tasks is with the sudo command, which grants elevated privileges on a temporary, per-command basis.

Shell Environment & Customization

Your command-line experience is shaped by several configuration elements:

  • Shell Startup Files: When a shell starts, it reads files like /etc/profile (for global settings) and ~/.bashrc (for personal settings) to configure the environment.
  • Customization: These files are used to set up your prompt, define aliases (shortcuts for commands), and set environment variables.
  • Environment Variables: These are named values that store information for the shell and other programs to use, such as your default text editor or command search path.
  • History: The history command lets you view, edit, and re-run previous commands. You can also cycle through them with the arrow keys.
  • Shortcuts: The command line has many keyboard shortcuts to make your work more efficient.

File Permissions & Ownership

Every file and directory in Linux has an owner, a group, and a set of permissions.

  • chmod: Changes a file's permissions (who can read, write, or execute it).
  • chown: Changes a file's owner.
  • chgrp: Changes a file's group.
