File Operation

Filesystem Varieties

Linux is very flexible and supports many different types of filesystems, which are chosen based on needs like performance, reliability, and the hardware used.

These can be grouped into several categories:

  • Native Linux: Filesystems created for Linux, like ext4 and btrfs.
  • Other Operating Systems: Compatibility with filesystems from Windows (ntfs, exfat), macOS (hfs+), and others.
  • Network: Filesystems that let you access files stored on other computers over a network, such as NFS.

The most advanced and commonly used types are journaling filesystems (ext4, xfs, btrfs), which are high-performance and very resistant to data corruption.

Linux Partitions

In Linux, a hard drive is typically divided into separate sections called partitions, with each partition holding a filesystem. Think of it like creating separate rooms in a large house.

This is done to organize data. For example, essential system files are often on their own root (/) partition, while user files (documents, downloads, etc.) are on a separate /home partition.

This separation has two major benefits:

  1. Stability: If one partition fills up (like your /home partition running out of space), it won't crash the entire operating system.
  1. Security & Safety: If a partition gets corrupted or compromised, the damage is contained to that single area, making it easier to fix.

Mount Points

Before you can use a new filesystem (like from a USB drive or a new partition), you have to attach it to your main directory structure. This process is called mounting.

The specific directory where you attach the filesystem is called a mount point. You may need to create this directory before you can mount anything to it.

Mounting and Unmounting

You use the mount command to attach a filesystem to a directory (mount point). Typically, you need sudo to run it.

  • To mount: sudo mount /dev/device /path/to/directory
user@lfs101: ~
$ sudo mount /dev/device /path/to/directory
  • To unmount: sudo umount /path/to/directory

If you want a filesystem to be mounted automatically every time you boot your computer, you must add it to a special configuration file called /etc/fstab (which stands for "filesystem table").

Note: The command to unmount is umount, not unmount.

NFS and Network Filesystems

A network filesystem allows you to access and share files across different computers over a network as if they were stored locally.

A common use case is to store all user home directories on a central server. This allows users to log into any client machine on the network and always see the same personal files and settings.

The two most common types are:

  • NFS (Network File System): The classic standard in the Linux/UNIX world.
  • CIFS/SAMBA: Commonly used for sharing files with Windows machines.

The Client-Server Architecture of NFS(based on the original from www.ibm.com)

NFS on The Server

Setting up an NFS server involves a few key steps:

Start the Service: Use sudo systemctl start nfs to start the NFS process. (Note: On systems like Fedora/RHEL, the service is called nfs-server).

user@lfs101: ~
$ sudo systemctl start nfs

Configure Shares: Edit the /etc/exports file to define which directories you want to share. Each line specifies a directory, who can access it, and their permissions, like this: /projects *.example.com(rw) This shares the /projects folder with read/write access to all computers in the example.com domain.

user@lfs101: ~
$ cat /etc/exports

Apply Changes: After editing /etc/exports, run exportfs -av to update the list of shared folders without interrupting the service.

user@lfs101: ~
$ sudo exportfs -av

Enable on Boot: To make NFS start automatically when the server boots, use sudo systemctl enable nfs.

user@lfs101: ~
$ sudo systemctl enable nfs

NFS on The Client

To connect to an NFS share from a client machine, you have two options: 💻

Temporary Mount: For a one-time connection, use the mount command directly. This mount will disappear when you reboot. sudo mount servername:/remote/share /local/directory

user@lfs101: ~
$ sudo mount servername:/remote/share /local/directory

Permanent Mount: To connect automatically every time you boot, you must add an entry to your /etc/fstab file. The line will look something like this: servername:/remote/share /local/directory nfs defaults 0 0

user@lfs101: ~
$ servername:/remote/share /local/directory nfs defaults 0 0

Pro Tip: When editing /etc/fstab for a network share, it's a good idea to add the nofail option. This prevents your computer from getting stuck during startup if the NFS server happens to be offline.

Overview of User Home Directories

In Linux, every user gets a personal home directory to store their files.

  • For regular users, their home directory is located inside /home (e.g., /home/username).
  • The system administrator, known as the root user, has a special, separate home directory located at /root.

On larger systems, the /home directory can be organized into subdirectories to group users (e.g., /home/students/, /home/faculty/).

The /dev Filesystem

The /dev directory contains special files known as device nodes, which are used to represent and interact with your system's hardware (like hard drives and printers).

This directory is not stored on the disk. Instead, a system called udev dynamically creates these files in memory as it detects devices when your computer boots up or when you plug something in.

Examples include:

  • /dev/sda1: The first partition on your hard drive.

Using diff3 and patch

The diff3 command is a tool for comparing three files at the same time.

Its most common use case is when two people have made separate changes to the same original file. diff3 can then compare the two new versions against the original to show how they both differ.

user@lfs101: ~
$ diff3 MY-FILE COMMON-FILE YOUR-FILE

A patch file is a file that contains only the differences between two versions of a file. It's an efficient way to distribute updates without sending the entire new file.

The patch command reads one of these patch files and applies the changes to your old version, bringing it up-to-date. These patch files are created using the diff command.

user@lfs101: ~
$ patch -p1 &lt; patchfile
$ patch originalfile patchfile

Using The file Utility

In Linux, file extensions (like .txt or .sh) are mostly for human convenience and do not reliably tell the system what type of file it is. Unlike Windows, a file named program.txt could still be an executable.

Backing Up and Compressing Data

Backing Up Data

While both cp and rsync can be used for backups, rsync is a much smarter and more efficient tool for the job.

Here are the key differences:

  • Efficiency: rsync checks if a file already exists at the destination and is unchanged. If so, it skips the copy. Furthermore, it only transfers the changed parts of a file, not the whole thing, which saves a lot of time. In contrast, cp just copies everything blindly.
  • Network Capability: rsync is designed to easily copy files between computers over a network. cp is generally limited to copying files on the local machine.

The -r option is commonly used with rsync to copy an entire directory and all its contents.

Using rsync

rsync is a very powerful utility for synchronizing and backing up files, often to a remote machine.

However, it can also be very destructive. A mistake in the command can easily overwrite or delete the wrong files.

Safety Tip: It is highly recommended to always test your rsync command first with the --dry-run option. This will show you exactly what the command would do without actually making any changes, so you can confirm it's correct.

user@lfs101: ~
$ rsync -r project-X archive-machine:archives/project-X
user@lfs101: ~
$ rsync --progress -avrxH  --delete sourcedir destdir

Compress Data Using gzip

gzip has historically been the most widely used Linux compression utility. It compresses well and is very fast.

CommandUsage
gzip *Compresses all files in the current directory; each file is compressed and renamed with a .gz extension.
gzip -r projectXCompresses all files in the projectX directory, along with all files in all of the directories under projectX.

Compress Data Using bzip2

bzip2 has a syntax that is similar to gzip but it uses a different compression algorithm and produces significantly smaller files, at the price of taking a longer time to do its work. Thus, it is more likely to be used to compress larger files.

CommandUsage

The bzip2 compression tool is now considered deprecated because it is no longer maintained and the newer xz tool offers better compression.

You should avoid using bzip2 for compressing new files. However, you will still need it to be able to decompress older files that you encounter with the .bz2 extension.

Compress Data Using xz

xz is a modern compression tool that is the most space-efficient of the common Linux utilities, meaning it creates the smallest files.

Compressed files are stored with a .xz extension.

Handling Files Using zip

The zip program is rarely used by Linux users to create new compressed files. It is considered a legacy tool that is not very fast or efficient.

Its main purpose in Linux is for compatibility: you'll need it to open and decompress .zip files that you get from users on other operating systems, like Windows.

CommandUsage
zip backup *Compresses all files in the current directory and places them in the backup.zip.
zip -r backup.zip ~Archives your login directory (~) and all files and directories under it in backup.zip.
unzip backup.zipExtracts all files in backup.zip and places them in the current directory.

Archiving and Compressing Data Using tar

The name tar originally stood for "tape archive" because it was used to back up files to magnetic tape.

Today, it's used to bundle multiple files and directories into a single archive file, often called a "tarball." A key feature of tar is that it can compress the archive as it's being created and decompress it during extraction, all in one command.

CommandUsage
tar xvf mydir.tarExtract all the files in mydir.tar into the mydir directory.
tar zcvf mydir.tar.gz mydirCreate the archive and compress with gzip.
tar jcvf mydir.tar.bz2 mydirCreate the archive and compress with bz2.
tar Jcvf mydir.tar.xz mydirCreate the archive and compress with xz.
tar xvf mydir.tar.gzExtract all the files in mydir.tar.gz into the mydir directory.NOTE: You do not have to tell tar it is in gzip format.

When using the tar command, the dash (-) before the options (like -cvf) is common but often not required.

While it's possible to run tar and a compression command as two separate steps, it is not recommended. This method is slower and wastes space by creating a large, uncompressed .tar file on your disk temporarily. It's better to do both actions in a single tar command.

user@lfs101: ~
$ tar cvf mydir.tar mydir ; gzip mydir.tar
user@lfs101: ~
$ tar cvf mydir.tar mydir
$ gzip mydir.tar

Disk-to-Disk Copying (dd)

The dd command is a very powerful low-level tool for making exact copies of raw disk space, such as backing up the Master Boot Record (MBR).

WARNING: dd is extremely dangerous and can easily wipe out an entire hard drive with a single typo. It does not ask for confirmation before overwriting data. Because of this, it is often jokingly called "disk destroyer." Be extremely careful when using this command.

user@lfs101: ~
$ dd if=/dev/sda of=/dev/sdb
Do not experiment with this command as written above, as it can erase a hard disk!

Summary

Filesystem Structure & Mounting

The Linux filesystem is a tree-like structure that starts at the root directory (/). Its layout is standardized by the Filesystem Hierarchy Standard (FHS) to ensure consistency across different distributions.

  • Partitions: Hard drives are divided into partitions to segregate files by type and purpose, which improves stability and organization.
  • Mounting: To be used, a filesystem on a partition must be attached to the main tree at a directory called a mount point.
  • /etc/fstab: This file is used to configure which filesystems are mounted automatically when the system boots.
  • NFS: The Network File System is a common protocol for sharing directories and files across a network.

Important Directories

Several directories have specialized purposes:

  • /proc: A pseudo-filesystem that exists only in memory and provides a window into the running kernel.
  • /root: The personal home directory for the root user (administrator).
  • /var: Contains variable data like logs and caches. It's often placed on its own partition to prevent its growth from filling up the entire system.
  • /boot: Holds the essential files needed to start up the Linux operating system, including the kernel itself.

Core File Utilities

A variety of powerful command-line tools are available for managing files.

  • File Extensions: Unlike in some other operating systems, file extensions in Linux do not reliably determine a file's type.
  • patch: Applies a "patch file," which contains only the differences (deltas) between an old version of a file and a new one.
  • cp vs. rsync: cp is used for simple copies on the local machine. rsync is a more powerful tool that can efficiently synchronize files both locally and between different computers over a network.
  • dd: A low-level tool for making exact, raw copies of data, which can include entire disk partitions. It is extremely powerful and must be used with caution.

Archiving & Compression

  • Compression Tools: Utilities like gzip, bzip2, and xz are used to reduce file sizes. zip is also available, primarily for compatibility with other operating systems.
  • tar: This command is used to bundle multiple files and directories into a single archive file, often called a tarball. It can work in conjunction with compression tools to create compressed archives (like .tar.gz) in a single step.
