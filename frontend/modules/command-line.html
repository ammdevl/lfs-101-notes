Command Line Operations

Launching Terminal Window

There are several ways to open a terminal in the GNOME desktop environment: >_

  • Applications Menu: Find and click the "Terminal" icon in your list of applications.
  • Desktop Right-Click: Right-click on your desktop and select "Open in Terminal."
  • Run Command: Press CTRL+ALT+T, type gnome-terminal, and press Enter.

Since the terminal can sometimes be hidden in menus, it's a good idea to add it to your "Favorites" or pinning it to your panel for quick access.

sudo

The sudo command allows a permitted user to run a command with the administrative privileges of another user, usually the root (or superuser) account.

Think of it as temporarily borrowing the administrator's power for a single command. On some distributions like Ubuntu, sudo is configured for the main user during installation, while on others, it must be set up manually.

Example of Using sudo

user@lfs101: ~
$ make me a sandwich
what? make it yourself
$ sudo make me a sandwich
okay

Turning Off the Graphical Desktop

The exact method for stopping the graphical desktop (GUI) in Linux can vary, but there are two main commands for it.

  • On modern distributions that use systemd, the GUI is a service that can be stopped with the systemctl utility.
  • Many distributions also support the older telinit command to achieve the same result.

Killing the GUI

user@lfs101: ~
$ sudo systemctl stop gdm
user@lfs101: ~
$ sudo telinit 3

Bringing The GUI Back

user@lfs101: ~
$ sudo systemctl start gdm
user@lfs101: ~
$ sudo telinit 5

Understanding Absolute and Relative Paths

Example Absolute Path

An absolute pathname begins with the root directory (/) and follows the tree, branch by branch, until it reaches the desired directory or file. Absolute paths always start with /.
user@lfs101: ~
$ cd /usr/bin

Example of Relative Path

A relative pathname starts from the present working directory. Relative paths never start with /.
user@lfs101: ~
$ cd ../../usr/bin

touch

touch is often used to set or update the access, change, and modify times of files. By default, it resets a file's timestamp to match the current time.

user@lfs101: ~
$ touch -t 12091600 myfile
This sets the myfile file&#x27;s timestamp to 4 p.m., December 9th (12 09 1600).

mkdir and rmdir

mkdir is used to create a directory:

user@lfs101: ~
$ mkdir sampdir
It creates a sample directory named sampdir under the current directory.
user@lfs101: ~
$ mkdir /usr/sampdir
It creates a sample directory called sampdir under /usr.

Removing a directory is done with rmdir. The directory must be empty or the command will fail.

user@lfs101: ~
$ rmdir sampdir

To remove a directory and all of its contents you have to do rm -rf.

user@lfs101: ~
$ rmdir sampdir
rmdir: failed to remove &#x27;my_stuff/project_folder&#x27;: Directory not empty
$ rm -rf sampdir

Moving, Renaming or Removing a File

Note that mv does double duty, in that it can:

  • Simply rename a file
  • Move a file to another location, while possibly changing its name at the same time.

If you are not certain about removing files that match a pattern you supply, it is always good to run rm interactively (rm –i) to prompt before every removal.

CommandUsage
mvRename a file | Move a file
rmRemove a file
rm -fForcefully remove a file
rm -iInteractively remove a file

Examples

mv does double duty.

user@lfs101: ~
$ mv samp.txt /Document/Text_Files
$ mv samp.txt samp1.txt

With rm , the file is simply deleted.

user@lfs101: ~
$ rm samp.txt

If you use rm -f , the file is deleted immediately and silently, with no questions asked, even if it was write-protected.

user@lfs101: ~
$ rm -f samp.txt

If you use rm -i , you would type y and press Enter to confirm the deletion. If you typed n, the file would not be deleted.

user@lfs101: ~
$ rm -i samp.txt
rm: remove regular empty file &#x27;important_file.txt&#x27;? y

I/O Rediration

Command shell redirection allows you to change input sources and output destinations for the three standard file streams (stdin, stdout, stderr), enabling input from files instead of keyboards and directing output to files or other commands.

For a program called do_something that uses standard streams, you can redirect input using the less-than sign (<):

user@lfs101: ~
$ do_something &lt; input-file

To send output to a file, use the greater-than sign (>):

user@lfs101: ~
$ do_something &gt; output-file

You can combine both redirections:

user@lfs101: ~
$ do_something &lt; input-file &gt; output-file

Since stderr is separate from stdout, error messages will still display on screen in the above example.

To redirect stderr to its own file, use the file descriptor number 2:

user@lfs101: ~
$ do_something 2&gt; error-file

NOTE: do_something 1> output-file equals do_something > output-file.

To send stderr to the same destination as stdout, use: 2>&1

user@lfs101: ~
$ do_something &gt; all-output-file 2&gt;&amp;1

bash offers a simpler syntax:

user@lfs101: ~
$ do_something &gt;&amp; all-output-file

Pipe

The UNIX/Linux philosophy favors simple, specialized programs working together rather than complex all-in-one applications. Pipes enable this collaboration by connecting programs, with the output of one becoming the input for another.

To create a pipe, use the vertical-bar symbol (|) between commands:

user@lfs101: ~
$ command1 | command2 | command3

Locate

The locate utility program performs a search by using a previously constructed database of files and directories on your system. It matches all entries containing a specified character string, which can sometimes result in a very long list.

To get a shorter and more relevant list, we can use the grep program as a filter. grep will print only the lines that contain one or more specified strings, as in:

user@lfs101: ~
$ locate txt | grep Documents
/home/username/Documents/samp1.txt
/home/username/Documents/samp2.txt
/home/username/Documents/samp3.txt
/home/username/Documents/samp4.txt

Wildcards and Matching Filenames

You can search for a filename containing specific characters using wildcards.

WildcardUsage
?Matches any single character
*Matches any string of characters
[set]Matches any character in the set of characters, for example [adf] will match any occurrence of a, d, or f
[!set]Matches any character not in the set of characters

Examples

To search for files using the ? wildcard, replace each unknown character with ?. For example, if you know only the first two letters ("ba") of a three-letter filename with an extension of .out, you would type ls ba?.out.

user@lfs101: ~
$ ls ba?.txt

To search for files using the * wildcard, replace any unknown string of characters with *. For example, if you only remember that the file has a .out extension, you can type ls *.out.

user@lfs101: ~
$ ls *.txt

The find Program

find is a powerful utility for Linux administrators. It searches through the filesystem tree from any directory to locate files matching specific conditions. By default, it searches from the current working directory.

Administrators commonly use it to locate old, large core files (containing program failure diagnostics) for removal.

/usr/share/doc/gcc-13-base/gcc

Using the Advanced find Options

A powerful feature of find is its ability to execute commands on files matching your search criteria. This is accomplished using the -exec option.

For example, you can find and remove all files that end with .swp:

user@lfs101: ~
$ find -name &quot;*.swp&quot; -exec rm {} &#x27;;&#x27;

Finding Files Based on Time and Size

You may often need to find files based on attributes such as creation date, last access time, or size. Performing such searches is straightforward.

To find files based on time:

user@lfs101: ~
$ find / -ctime 3

To find files based on sizes:

user@lfs101: ~
$ find / -size 0

For example, to find files greater than 10 MB in size and running a command on those files:

user@lfs101: ~
$ find / -size +10M -exec command {} ’;’

Installing Software

Package Management Systems on Linux

Linux distributions and their add-on software are installed through Package Management Systems. Each package includes files and instructions for a software component to work properly with other system components. Packages often have dependencies; for instance, a Python web application package will need specific Python packages installed.

Two main package manager families exist: Debian-based and those using RPM. Though incompatible, both provide similar core features and meet the same needs. Some specialized Linux distributions use alternative systems.

Package Managers: Two Levels

Package management systems operate on two distinct levels:

  1. A low-level tool (such as dpkg or rpm) handles unpacking individual packages, running scripts, and installing software correctly.
  1. A high-level tool (such as apt, dnf, or zypper) manages groups of packages, downloads them from vendors, and resolves dependencies.

Users typically only need to work with the high-level tool, which automatically calls the low-level tool when necessary. Dependency resolution is one of the high-level tool's most valuable features, as it finds and installs all required dependencies for you. Be cautious, though—installing a single package might trigger the installation of dozens or even hundreds of dependent packages.

Working With Different Package Management Systems

apt is the package management system for Debian-based systems. It powers graphical interfaces like Ubuntu Software Center and synaptic, with command-line tools including apt (or apt-get) and apt-cache.

dnf is the command-line package manager for RPM-compatible Linux systems in the Red Hat family.

Summary

Terminals & System Control

Interaction with the Linux command line happens through terminals.

  • Types of Terminals: You can use full-screen Virtual Terminals (VTs), or you can use a terminal emulator, which is a program that provides a command line inside a window on your graphical desktop.
  • Password Entry: For security, when you type your password in the terminal, no characters will appear on the screen.
  • Shutdown: The preferred and safest way to shut down or reboot the system from the command line is with the shutdown command.

Navigating the Filesystem

Understanding how to specify file locations is crucial.

  • Paths: There are two types of pathnames:
    • An absolute path starts from the root directory (/) and spells out the full location (e.g., /home/user/Documents).
    • A relative path starts from your current location (e.g., Documents/file.txt).
  • Links: Hard and soft (symbolic) links are useful for creating shortcuts and references to files and directories.
  • cd -: This is a handy shortcut that takes you back to the directory you were in previously.

Finding & Managing Files

Linux provides powerful tools for locating and managing files.

  • locate vs. find: These are the two primary search tools.
    • locate: Performs a very fast search using a pre-built database of all files on the system.
    • find: Performs a slower but more flexible real-time search, looking recursively through a specific directory. It can also execute commands on the files it finds using the exec option.
  • touch: A simple utility used to create new, empty files or to update the modification and access timestamps of existing files.

Software Package Management

Different Linux distribution families use their own command-line tools to install, update, and remove software.

  • apt: The Advanced Packaging Tool used by Debian-based systems (like Ubuntu and Mint).
  • dnf: The package manager used by modern Red Hat-based systems (like Fedora, CentOS, and RHEL).
  • zypper: The package manager used by openSUSE.
