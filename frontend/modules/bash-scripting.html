The bash Shell and Basic Scripting

Features and Capabilities

Shell Scripting

A shell script is a simple text file containing a series of commands that the command line can execute. The primary purpose of a shell script is automation, especially for repetitive tasks.

Instead of typing the same sequence of commands over and over, you can save them in a script and run it with a single command. This saves time, reduces errors, and ensures consistency.

In Linux, these scripts are most commonly written to be run by the bash shell interpreter.

Command Shell Choices

Interpreters & Benefits of Scripting

A command interpreter is the program that reads and executes the commands contained within a shell script. While many interpreters exist for different languages (like Python or Perl), for shell scripting, this refers to a command shell like bash.

Using scripts offers several key benefits over manually typing long sequences of commands:

  • Automation: Scripts make complex and repetitive tasks quick and efficient.
  • Reliability: They reduce the potential for human error.
  • Reusability: Because scripts are saved as files, they can be easily reused, modified, and shared with other users.

Available Shells

Linux supports a wide variety of command shells. You can see a list of the shells available on your system by viewing the contents of the /etc/shells file. Common choices include sh, csh, ksh, and zsh.

However, the most common and widely used shell is bash (the Bourne-Again Shell), and it serves as the default for the vast majority of Linux users.

Shell Scripts

A shell is a command-line interpreter that you can either use interactively in a terminal or to run a shell script. Running a script is simply an automated way of typing the commands contained within it.

The first line of any script is very important. It must begin with the #! characters, which are known as a shebang. The shebang is followed by the full path to the command interpreter that should be used to execute the script. It essentially tells the system, "Run this file using this specific program."

While this is often /bin/bash for shell scripts, it can be any interpreter, such as /usr/bin/python or /usr/bin/perl, making the shebang a universal feature for creating executable scripts in any language.

A Simple bash Script

Step 1: Write the Script

First, create a simple text file named hello.sh using a text editor or the cat command. The file should contain two lines: a shebang that specifies the bash interpreter and an echo command to print a message.

Bash

user@lfs101: ~
$ cat &gt; hello.sh
#!/bin/bash
echo &quot;Hello Linux Foundation Student&quot;

Step 2: Make it Executable

Before you can run the script directly, you need to give it execute permission. Use the chmod command to do this:

user@lfs101: ~
$ chmod +x hello.sh

Step 3: Run the Script

There are two common ways to execute your script:

  • Directly: Type ./hello.sh. This method requires the file to have execute permission (from Step 2).
  • With the Interpreter: Type bash hello.sh. This method tells the bash interpreter to run the commands in the file, so it does not require execute permission.
user@lfs101: ~
$ ./hello.sh
Hello Linux Foundation Student

Interactive Example Using bash Scripts

You can make your scripts interactive by prompting a user for input and then using that input. This is primarily done with the read command.

The Key Commands

  • echo: Used to display a message on the screen, typically to prompt the user for what they should enter.
  • read variable_name: This command pauses the script and waits for the user to type something and press Enter. The text entered by the user is then stored in the specified variable.
  • $variable_name: To use the value stored in a variable, you place a dollar sign ($) in front of its name.

Example Interactive Script

Here is an example script named getname.sh that asks for a user's name and then displays it. The lines starting with # are comments, which are ignored by the interpreter.

user@lfs101: ~
$ cat &gt; getname.sh
#!/bin/bash
# Prompt the user for their name
echo &quot;ENTER YOUR NAME&quot;

# Read the user&#x27;s input and store it in the &#x27;name&#x27; variable
read name

# Display the stored value
echo &quot;The name given was: $name&quot;

How to Run It

  1. First, make the script executable using the chmod command: chmod +x getname.sh
  1. Then, run the script from your terminal: ./getname.sh

The script will then prompt you for your name, wait for your input, and finally echo the name you entered back to the screen.

user@lfs101: ~
$ chmod +x getname.sh
$ ./getname.sh
ENTER YOUR NAME
Aung Myint Myat
The name given was: Aung Myint Myat

Return Values

Every shell script produces a numerical return value (or exit code) when it finishes executing. This code acts as a signal to the system, indicating whether the script succeeded or failed.

You can manually set this value from within your script using the exit statement (e.g., exit 0 for success or exit 1 for an error). This mechanism is essential for automation, as it allows a parent process or another script to check the outcome of the script it ran and decide what to do next.

Viewing Return Values

After a command or script finishes, it provides a numerical return value (also known as an exit code) to indicate whether it was successful. The convention is simple and universal:

  • 0 means Success.
  • Any non-zero value means Failure.

You can check the exit code of the most recently executed command by viewing the special variable echo $?.

For example, running ls on a file that exists will result in an exit code of 0. Running it on a file that doesn't exist will result in a non-zero exit code (like 2). Different non-zero codes can represent different types of errors, and their meanings are often documented in the command's man page.

Splitting Long Commands Over Multiple Lines

When a command is too long to be easily read on a single line, you can use the backslash (\) as a line continuation character to split it across multiple lines.

Placing a \ at the very end of a line tells the shell to ignore the line break and treat the next line as part of the same command. This is done purely for readability and does not change how the command is executed. The shell processes it as one single, long command.

Putting Multiple Commands on a Single Line

Command chaining is a technique for running multiple commands in a sequence from a single line. The operator you use between the commands determines how the chain behaves.

; (Semicolon): Run Sequentially

The semicolon (;) is the simplest chaining operator. It executes commands one after another, regardless of whether the previous command succeeded or failed.

Example: command1 ; command2 (Both command1 and command2 will always run).

&& (AND): Run on Success

The AND operator (&&) creates a conditional chain. It will only execute the next command if the previous command was successful (returned an exit code of 0). If any command fails, the rest of the chain is aborted. This is useful for tasks where each step depends on the previous one working correctly.

Example: make && make install (make install will only run if make succeeds).

|| (OR): Run on Failure

The OR operator (||) also creates a conditional chain, but it works the opposite way. It will only execute the next command if the previous command failed (returned a non-zero exit code). The chain stops as soon as a command succeeds. This is useful for creating fallbacks or trying alternative commands.

Example: cat file.txt || echo "File not found" (The echo command only runs if cat fails).

Chaining vs. Piping

It's important not to confuse chaining with piping (|).

  • Chaining: Each command runs and finishes completely before the next one starts. The logic is based on the exit status of the commands.
  • Piping: Commands run concurrently, and the data output of one command is fed directly as the data input to the next.

Output Redirection

Output redirection is the process of diverting a command's output from the screen (standard output) and sending it to a file instead. There are two main operators used for this.

> (Create / Overwrite)

The single greater-than sign (>) is used to write a command's output to a file.

  • If the file does not exist, it will be created.
  • If the file already exists, its contents will be overwritten with the new output.

Example: ls -l > file_list.txt

>> (Append)

The double greater-than sign (>>) is used to append a command's output to a file.

  • If the file does not exist, it will be created.
  • If the file already exists, the new output will be added to the end of the file without deleting its previous contents.

Example: echo "New log entry" >> system.log

Input Redirection

Just as output can be sent to a file, input redirection allows a command to take its input from a file instead of the keyboard. This is done using the less-than symbol (<).

For many commands, there are several equivalent ways to process a file. For example, the following three commands all use wc to count the lines, words, and characters in /etc/passwd and produce the exact same result:

  • wc < /etc/passwd (Input Redirection)
  • wc /etc/passwd (File Argument)
  • cat /etc/passwd | wc (Piping)

While all three methods work, using a direct file argument (when available) is often the simplest to read.

Build-In Shell Commands

A shell script can execute three different types of commands to perform its tasks.

Compiled Applications

These are standalone, executable programs that are compiled from a language like C and reside in standard system directories (like /usr/bin). A script has access to all these common tools, such as ls, rm, vi, and grep.

Shell Built-in Commands

These are commands that are part of the bash shell itself, such as cd, pwd, echo, and read. They can only be run from within a shell or a shell script.

It's important to know that a built-in command may share the same name as a compiled application (e.g., the built-in echo vs. the program /bin/echo) but behave slightly differently. You can see a list of all built-in commands by typing help in your terminal.

Other Scripts

A shell script is not limited to running only applications and built-ins; it can also execute other scripts. This includes calling another shell script or running a script written in an entirely different interpreted language, such as Python or Perl.

Script Parameters

You can pass input to a shell script by providing parameters (also called arguments) on the command line after the script's name. This allows the script to act on different data or change its behavior each time it's run.

Inside the script, these parameters are accessed using special variables called positional parameters:

  • The first argument is stored in $1.
  • The second argument is stored in $2.
  • ... and so on for $3, $4, etc.

For example, if you run the command ./myscript.sh hello 123, then inside the script, the value of $1 would be "hello" and the value of $2 would be "123".

user@lfs101: ~
$ ./myscript.sh
hello 123

Using Script Parameters

When you run a shell script, it has access to the command-line arguments through a set of special variables known as positional parameters. Here's what each one represents:

  • $0: The name of the script itself.
  • $1, $2, $3, etc.: The first, second, third, and subsequent arguments passed to the script.
  • $*: A single string containing all the arguments together.

For example, if you run the command ./myscript.sh hello world:

  • $0 would be ./myscript.sh.
  • $1 would be hello.
  • $2 would be world.
  • $* would be hello world.

Command Substitution

Command substitution is a powerful shell feature that allows you to use the output of one command as part of another command.

How it Works

The shell first executes the inner command in parentheses or backticks. It then captures that command's standard output and substitutes it into the place of the outer command before running it.

Modern Method: $(...)

The modern, recommended, and most powerful way to perform command substitution is by enclosing the command in $(...).

This method's key advantage is that it supports nesting, meaning you can place one command substitution inside another. It should always be used for new scripts.

Example: ls /lib/modules/$(uname -r)/ In this case, uname -r is run first, and its output (the kernel version) is inserted into the ls command's path.

Deprecated Method: Backticks (`...`)

An older method using backticks (e.g., `uname -r`) also exists. However, this syntax is deprecated. It is less readable, does not support nesting, and should be avoided in all new scripts and commands.

Environment Variables

Shell scripts use variables to store values that can be referenced throughout the script. These can be pre-existing environment variables (like HOME or PATH) or variables you define yourself.

The Golden Rule: Setting vs. Using Variables

There is a critical difference in syntax when you are setting a variable versus when you are using its value.

  • Setting a Variable: To assign a value, you use the format VARIABLE=value. Do not use a dollar sign ($). MYVAR="hello"
  • Using a Variable: To get the value stored in a variable, you must prefix its name with a dollar sign ($). echo $MYVAR

Viewing Variables

To see a list of all the environment variables currently set in your session, you can use commands like env, set, or printenv.

Exporting Variables

What export Does

By default, variables you create within a shell script are local, meaning they are only visible within that script. The export command is used to promote a local variable to an environment variable, making it visible to any child processes (or sub-shells) that the script launches.

You can either set and export a variable in one step (export VAR=value) or set it first and then export it (VAR=value; export VAR). Running export by itself will list all currently exported variables.

The Inheritance Rule: Copied, Not Shared

It's crucial to understand that exported variables are copied, not shared. A child process receives its own independent copy of the exported variable from its parent.

This means that if a child process modifies the value of an exported variable, the change is local to that child process only. The parent process's original variable will not be affected.

Functions

What is a Function?

A function (also called a subroutine) is a named, reusable block of code within a script that performs a specific set of operations. Functions are essential for organizing your code and avoiding repetition, as you can define a procedure once and then execute it multiple times.

How to Use Functions

Using a function is a two-step process:

  1. Declare the function: You define the function's name and the commands it will execute inside curly braces {}.
  1. Call the function: You run the code inside the function by simply typing its name as a command.

The basic syntax for declaring a function is:

user@lfs101: ~
function_name () {
    # commands go here
}

Passing Arguments to Functions

Functions can accept arguments, just like a shell script can. When you call a function, you can pass values to it.

Inside the function, these arguments are accessed using positional parameters: $1 refers to the first argument, $2 refers to the second argument, and so on.

The if Statement

What it Does

The if statement is a fundamental control structure that allows a shell script to make decisions and execute commands conditionally. It works by evaluating a condition; if the condition is true, one set of commands is executed, and if it's false, a different set can be run.

A condition can be based on several types of tests, including:

  • The exit code of a command (checking for success/failure).
  • Comparisons between numbers or text strings.
  • The properties of a file (e.g., does it exist? Is it readable?).

Basic Structure

The if statement has a simple but specific structure. The else block is optional.

user@lfs101: ~
if [ condition ]
then
    # These commands run if the condition is true.
else
    # These commands run if the condition is false.
fi

Using The if Statement

The test Command

In a shell script's if statement, the condition is usually evaluated using the test command, which is most often written using square brackets. This command performs a check and returns a success or failure exit code that the if statement uses to make its decision.

For example, the test [ -f "$1" ] uses the -f operator to check if the item passed as the first argument ($1) exists and is a regular file.

[ vs. [[: Choosing Your Brackets

There are two ways to write the test command, and the modern version is generally preferred.

  • [ ... ] (single brackets): This is the original, traditional test command. It is universally compatible with all shells.
  • [[ ... ]] (double brackets): This is a newer, more powerful version built into modern shells like bash. It is generally safer, more flexible, and less error-prone than single brackets (for example, it handles empty variables more gracefully). For scripts written specifically for bash, using double brackets is the recommended practice.
user@lfs101: ~
if [ -f &quot;$1&quot; ]
then
     echo file &quot;$1&quot; exists 
else
    echo file &quot;$1&quot; does not exist
 fi

The elif Statement

You can use the elif statement to perform more complicated tests, and take action appropriate actions. The basic syntax is:

user@lfs101: ~

Testing for Files

What You Can Test

You can use an if statement in bash with file conditionals to test for various file attributes. This allows your script to make decisions based on the state of a file. Common things you can check for include:

  • File or directory existence.
  • Read, write, or execute permissions.

Example & Syntax

A test is performed inside square brackets []. For instance, the operator -x tests if a file is executable.

user@lfs101: ~
if [ -x /path/to/file ] ; then
    # Action to perform if the file is executable
fi

It's a common and compact practice to write the then on the same line as the if statement, separated by a semicolon.

How to Learn More

Boolean Expressions

When you chain multiple conditions together, bash uses a logic called short-circuit evaluation to work more efficiently. It stops evaluating the chain as soon as the final outcome is known.

Boolean expressions evaluate to either TRUE or FALSE.

&& (AND) Operator

When using the && (AND) operator, the shell checks conditions from left to right and stops as soon as a condition is false. Since all conditions must be true for the entire chain to be true, there's no need to check the rest once a single false condition is found.

For example, in A && B && C, if B is false, C will never be evaluated.

|| (OR) Operator

When using the || (OR) operator, the shell checks conditions from left to right and stops as soon as a condition is true. Since only one condition needs to be true for the entire chain to be true, there's no need to check the rest once a single true condition is found.

For example, in A || B || C, if A is false andB is true, C will never be evaluated.

Tests In Boolean Expressions

What is a Boolean Expression?

A Boolean expression is a conditional test that evaluates to one of two results: TRUE or FALSE. In shell scripting, these expressions are the foundation of decision-making and are typically used within if statements to control a script's flow. They can be used to compare numbers, strings, or check the status of files.

Examples

Boolean tests are performed inside square brackets [] or [[]].

  • File Tests: You can check for file attributes. For example, the e operator checks if a file exists. [ -e /etc/hosts ]
  • Numeric Tests: You can compare numerical values. For example, the gt operator checks if one number is greater than another. [ $number1 -gt $number2 ]

Example of Testing Strings

You can use an if statement to compare two strings to check if they are identical. 👯 The standard operator for this is == (double equals).

The basic syntax is:

user@lfs101: ~
if [ &quot;$string1&quot; == &quot;$string2&quot; ] ; then
    # Action to take if the strings are equal
fi

For example, you can use this to check if a user's input matches an expected word.

Note: A = (single equals) sign will also work for string comparison inside single brackets [ ], but == is often preferred for clarity and is the standard in more modern [[ ]] tests.

Arithmetic Expressions

There are three main ways to perform arithmetic in a shell script, but the modern $(()) syntax is strongly preferred.

Modern Method: $((...))

The recommended way to perform arithmetic in modern shell scripts is with arithmetic expansion, using the $((...)) syntax. It is built directly into the shell, making it efficient and easy to read.

Example: result=$((10 + 5))echo $result

Older Methods

Two older methods also exist but are generally not recommended for new scripts.

  • expr: This is an external program with a strict syntax that requires spaces around operators (e.g., expr 10 + 5). It is now considered deprecated.
  • let: This is a built-in shell command with its own syntax (e.g., let result=10+5).

For all new scripts, you should use the $((...)) syntax for better performance, readability, and consistency.

Summary

A shell script is a file containing a sequence of commands that are executed by a command-line interpreter, most commonly bash in Linux.

Scripting Fundamentals

  • Shell Script: A series of commands saved in a file to automate tasks.
  • Functions: Reusable, named blocks of commands within a script that help organize code and avoid repetition.

Data & Variables

Scripts use variables and parameters to handle data.

  • Environment Variables: Named values that can be used by the shell and other programs to control their behavior.
  • export: The command used to make a variable visible to any child processes launched by the script.
  • Parameters: Values passed to a script from the command line to influence its execution.

Input/Output (I/O) & Flow Control

Scripts can control where they get input from, where their output goes, and what decisions they make.

  • Output Redirection: The process of sending a command's output to a file instead of the screen (e.g., using >).
  • Input Redirection: The process of making a command read its input from a file instead of the keyboard (e.g., using <).
  • if Statement: A fundamental control structure that allows a script to make decisions and execute code conditionally.

Expressions & Substitution

  • Command Substitution: A feature that allows you to use the output of one command as part of another command (e.g., $(command)).
  • Arithmetic Expressions: Combinations of numbers and operators (like +, , ) used for performing calculations within a script.
