Network Operations

Network Addresses and DNS

Introduction to Networking

A network is a group of computers and other devices (called nodes) connected together through channels like cables or wireless signals, allowing them to communicate and share resources.

The main purposes of a network are to:

  • Allow connected devices to communicate with each other.
  • Enable users to share hardware like printers, scanners, and servers.
  • Easily share and manage information like files and databases.

Most organizations use an internal network for local sharing and connect to the Internet, which is the largest "network of networks" in the world.

IP Addresses

Every device on a network needs a unique IP address, which acts like a mailing address for routing information. Data travels across a network by being broken down into small pieces called packets.

Each packet is like a small envelope containing:

  • A piece of the actual data being sent.
  • A header with information like the source and destination IP addresses and the packet's sequence number, so the data can be reassembled in the correct order.

    This system is complex because networking software must support a huge diversity of devices and many old standards.

IPv4 and IPv6

There are two versions of the Internet Protocol address in use today: the older IPv4, which is still the most common, and the newer IPv6, which was created to solve its predecessor's limitations.

IPv4: The Old Standard

IPv4 uses 32-bit addresses, which provides a total of about 4.3 billion unique addresses. While this once seemed like a large number, the rapid growth of internet-connected devices means that we're running out of available IPv4 addresses.

IPv6: The New Standard

To solve the address shortage, IPv6 was developed. It uses 128-bit addresses, which allows for 3.4×1038 unique addresses—a virtually limitless supply that can support the internet 's future growth for a very long time.

The Challenge of Migration

Despite the clear need for IPv6, the global transition has been slow. This is because migration is complex and the two protocols are not always able to communicate with each other seamlessly. Upgrading networks and equipment requires significant effort, so both IPv4 and IPv6 will continue to coexist for the foreseeable future.

Decoding IPv4 Addresses

IPv4 Address Structure

A 32-bit IPv4 address is broken down into four 8-bit sections called octets (an octet is another word for a byte). This is typically written as four decimal numbers separated by dots, such as 172.16.31.46.

Address Classes

IPv4 addresses are categorized into five classes: A, B, C, D, and E.

The most common classes (A, B, and C) are divided into two distinct parts:

  • Net ID (Network ID): This part identifies the specific network. Think of it as the street name.
  • Host ID: This part identifies a specific device (or host) on that network. Think of it as the house number.

The other two classes have special purposes:

  • Class D: Used for multicast applications (sending data to a group of devices at once).
  • Class E: Reserved for future or experimental use.

Class A Network Addresses

IPv4 Class A addresses were designed for a small number of very large networks, like those for major international corporations or early internet service providers.

Their structure is simple:

  • Net ID: The first octet (the first number) identifies the network.
  • Host ID: The last three octets identify the specific device on that network.

This design means there can only be a maximum of 126 Class A networks in total, but each one can support over 16 million hosts (devices). You can easily identify a Class A address because its first number will always be in the range of 1 to 127.

Class B Network Addresses

IPv4 Class B addresses represent a middle ground, designed for medium-to-large-sized networks like universities or large companies.

Their structure is evenly balanced:

  • Net ID: The first two octets identify the network.
  • Host ID: The last two octets identify the specific device on that network.

This design allows for 16,384 Class B networks, each supporting up to 65,536 hosts. You can identify a Class B address because its first number will always be in the range of 128 to 191.

Class C Network Addresses

IPv4 Class C addresses are the most common type, designed for a very large number of small local networks, such as those in homes and small businesses.

Their structure prioritizes the number of networks over the number of hosts:

  • Net ID: The first three octets identify the network.
  • Host ID: Only the last octet identifies the specific device on that network.

This design allows for nearly 2.1 million unique networks, but each network can only support a maximum of 256 hosts. You can identify a Class C address because its first number will always be in the range of 192 to 223.

IP Addresses Allocation

There are two primary methods for assigning IP addresses to devices on a network: manual (static) and automatic (dynamic).

Manual (Static) Assignment

In this method, a network administrator manually configures each device with a permanent, non-changing (static) IP address. This is typically done for important devices like servers, printers, or network hardware that need a predictable and consistent address.

Dynamic (DHCP) Assignment

This is the more common and automated method. A central server on the network uses the Dynamic Host Configuration Protocol (DHCP) to automatically assign temporary IP addresses to devices as they connect. These dynamic addresses can change over time, for example, each time a device reboots. This is how most computers and mobile devices get their IP addresses on office and home networks.

A Note on Home Networks (NAT)

In a typical home network that uses Network Address Translation (NAT), your router receives a single public IP address from your Internet Service Provider (ISP). The router then acts as a DHCP server, assigning private, internal IP addresses to all the devices connected to your home network.

Name Resolution

What is Name Resolution?

Name resolution is the process that converts a numerical IP address into a human-readable hostname. This is essential because hostnames are much easier for people to remember and use.

For example, the IP address 3.13.31.214 is resolved to the hostname linuxfoundation.org.

The hostname Command & localhost

You can interact with your system's hostname directly from the command line.

  • hostname: Typing this command with no arguments will display your computer's current hostname.
  • localhost: This is a special hostname that always refers to the machine you are currently on. Its corresponding IP address is always 127.0.0.1.

Note that trying to change your hostname by typing hostname <new-name> requires root privileges.

user@lfs101: ~
$ hostname
computer_name

Network Configuration Tools

Network Configuration Files

Historically, configuring network settings in Linux involved editing text files in the /etc directory, but the exact location of these files was inconsistent across different distributions (e.g., /etc/network/ on Debian vs. /etc/sysconfig/network on Red Hat).

The Modern Way: NetworkManager

To solve this problem, modern Linux systems have standardized on NetworkManager, a unified service for managing network connections. It provides a consistent backend and offers several user interfaces for configuration:

  • Graphical Tools (GUI): The easiest to use for desktop users, though their appearance can vary slightly between different desktop environments.
  • nmtui (NetworkManager Text User Interface): A simple, menu-driven tool that runs in your terminal. It's a great choice because it's easy to use and looks the same across all distributions.
  • nmcli (NetworkManager Command Line Interface): A powerful and scriptable command-line tool that is also consistent across all systems.

While you can use the graphical tools if you're comfortable with them, learning nmtui or nmcli is highly recommended if you work with different types of Linux systems, as they provide a consistent experience everywhere.

Network Interfaces

What is a Network Interface?

A network interface is the connection point between your computer and a network. It can be a physical piece of hardware, like a Network Interface Card (NIC), or it can be a virtual, software-based interface. A system can have multiple interfaces running at once, and each can be individually activated ("brought up") or deactivated ("brought down").

Viewing Interface Information: ip vs. ifconfig

There are two primary command-line tools for viewing information about your network interfaces.

  • ifconfig: This is the older, traditional tool. While its output is often considered easier for humans to read, it is now deprecated and has fewer capabilities. Many modern Linux distributions do not install it by default, so you may need to manually install the net-tools package to use it.
  • ip: This is the newer, more powerful, and currently recommended tool for all network-related tasks in Linux. It has far more capabilities than ifconfig, even though its output can sometimes seem less straightforward. It is the modern standard for network configuration and inspection.

The ip Utility

The ip command is the modern, powerful, all-in-one utility for viewing and managing network settings in Linux. It's designed to replace a collection of older, separate commands like ifconfig and route.

Two of its most common uses are:

  • To view IP addresses on all interfaces: ip addr show
  • To view the system's routing table: ip route show
user@lfs101: ~
$ ip addr show

$ ip route show

ping

The ping command is a simple utility used to check if a remote computer is online and reachable over the network. It works like sonar, sending a small data packet (a "ping") and waiting for a reply (a "pong").

By default, ping will run continuously. You can stop it by:

  • Pressing CTRL+C to stop it manually.
  • Using the c (count) option to send a specific number of packets (e.g., ping -c 4 google.com).

Note that some hosts are configured to ignore ping requests for security reasons, so a lack of response doesn't always mean the machine is offline.

route

Data travels across a network by being passed between a series of routers in a process of "hops"; it doesn't go straight from the source to the destination.

Each router decides where to send the data next by consulting its routing table, which acts like a map for the network. This table correlates a final destination with the address of the next hop on the path.

You can view your system's routing table using either the older route command or the newer, more powerful ip route command. These tools can also be used by an administrator to manually add or change "static" routes.

TaskCommand
Show current routing table$ route –n or ip route
Add static route$ route add -net address or ip route add
Delete static route$ route del -net address or ip route del

traceroute

The traceroute command is a network diagnostic tool that shows you the exact route (or path) your data takes to reach a destination host. It lists every router (or "hop") the packet passes through along the way, essentially giving you a turn-by-turn map of its journey across the internet.

This is extremely useful for troubleshooting network problems. By showing the time taken to reach each hop, you can pinpoint where delays or connection failures are occurring on the network. The basic syntax is traceroute <hostname_or_IP>.

Browsers, wget and curl

Graphical and Non-Graphical Browsers

Browsers are used to retrieve, transmit, and explore information resources, usually on the World Wide Web. Linux users commonly use both graphical and non-graphical browser applications.

The common graphical browsers used in Linux are:

  • Firefox
  • Google Chrome
  • Chromium
  • Konqueror
  • Opera

Sometimes, you either do not have a graphical environment to work in (or have reasons not to use it) but still need to access web resources. In such a case, you can use non graphical browsers, such as the following:

  • lynx (configurable text-based web browser; the earliest such browser and still in use)
  • elinks (based on lynx; it can display tables and frames)
  • w3m (another text-based web browser with many features)

wget

wget is a powerful command-line utility for downloading files from the internet, especially when a graphical browser is impractical. It's ideal for scripting and for handling complex download jobs.

Key features include:

  • Handling large file downloads.
  • Performing recursive downloads to grab entire websites by following links.
  • Working with password-protected sites.
  • Downloading multiple files at once.

The basic syntax is as simple as wget <url>.

user@lfs101: ~
$ wget url

curl

curl is a powerful command-line tool used to transfer data from or to a server using URLs. It's often used to quickly view a web page's source code or test an API directly from the terminal.

It has two primary modes of operation:

  • Display content: Simply typing curl <URL> will fetch the data and display it directly in your terminal.
  • Save to a file: Using the o option, you can save the content to a file, similar to wget. For example: curl -o page.html http://example.com.

Transferring Files

FTP (File Transfer Protocol)

What is FTP?

FTP (File Transfer Protocol) is a classic method for transferring files between computers over a network, built on a client-server model. A user runs an FTP client to connect to an FTP server to upload or download files.

The Problem with FTP

FTP is a very old protocol, dating back to the 1970s. Its biggest drawback is that it is fundamentally insecure. All data, including usernames and passwords, is sent in plain text without encryption, making it easy to intercept.

Because of these security flaws, FTP is considered inadequate for modern needs, and many websites have stopped using it. It is sometimes still used for anonymous FTP, where you are downloading public files and security is not a concern.

FTP Clients

FTP Clients

FTP clients are applications that allow you to transfer files using the FTP protocol. There are several types:

  • Graphical Clients: Programs like Filezilla provide an easy-to-use, drag-and-drop interface.
  • Web Browsers: Most browsers can access FTP servers by using a URL that starts with ftp://.
  • Command-line Tools: Utilities like ftp, ncftp, and the more secure sftp are available for use in the terminal.

FTP vs. SFTP: The Security Difference

It's crucial to understand the security implications of different file transfer protocols.

  • FTP (File Transfer Protocol): Standard FTP is an old protocol and is now considered insecure. It transmits all data, including usernames and passwords, in plain text without encryption, making them easy for an attacker to intercept. Because of this, modern tools like rsync and https are often preferred.
  • SFTP (SSH File Transfer Protocol): This is the secure, modern alternative. SFTP uses the Secure Shell (SSH) protocol to encrypt the entire connection, including your login credentials and the files being transferred. This makes it a much safer choice for handling sensitive information. However, unlike FTP, SFTP does not support anonymous (guest) access.

SSH: Executing Commands Remotely

What is SSH?

SSH (Secure Shell) is a cryptographic network protocol used to create a secure, encrypted connection to another computer over a network. Its primary purpose is remote administration, allowing you to securely manage a system from anywhere as if you were physically present.

How to Use SSH

SSH can be used in several ways from your command line:

  • Interactive Login: To log in and start a full command-line session on the remote machine, use the format: ssh username@hostname You will then be prompted for your password. (It's also possible to configure a more secure, passwordless login using SSH keys).
  • Running a Single Command: If you just need to run one command without starting a full session, you can add it to the end: ssh username@hostname "command to run" The command will execute on the remote machine, and the output will be displayed on your local terminal.

Copying Files Securely With scp

The scp (Secure Copy) command is used to securely copy files between two computers over a network. It uses the SSH protocol for the data transfer, which means the entire file and the login credentials are fully encrypted.

The syntax is similar to the standard cp command, but includes the remote host's information. For example, to copy a local file to a remote system, you would use: scp local_file.txt username@remote_host:/path/to/destination

By default, scp will prompt for a password, but it can also be configured to use passwordless SSH keys for convenience.

user@lfs101: ~
$ scp samp.txt username@hostname:/home/username/Downloads

Summary

Core Concepts: IP Addressing & DNS

Every device on a network is assigned a unique IP (Internet Protocol) address. This address has two parts: one that identifies the network and one that identifies the specific host (device) on that network.

  • Versions: There are two versions of IP addresses in use: the older IPv4 (32-bit) and the newer IPv6 (128-bit), which provides a vastly larger number of available addresses.
  • DNS: The Domain Name System is the internet's phonebook. It translates human-friendly domain names (like linuxfoundation.org) into their corresponding numerical IP addresses.

Network Diagnostics & Monitoring

Linux provides a powerful set of command-line tools to inspect and troubleshoot network connections.

  • ping: Checks if a remote host is online and responding by sending a small data packet and waiting for a reply.
  • ip addr show: The modern command to view the IP addresses assigned to your network interfaces.
  • ip route show: The modern command to view the system's IP routing table.
  • ifconfig and route: These are the older, traditional commands for viewing interface and routing information, respectively. They have been largely replaced by the more powerful ip command.

Web & Download Tools

A variety of tools are available for accessing content on the web.

  • Web Browsers: Linux supports both graphical browsers (like Firefox, Google Chrome) and lightweight text-based browsers (like lynx and w3m) that run in the terminal.
  • wget: A command-line utility designed for downloading files and entire websites.
  • curl: A versatile tool for transferring data to or from a server using a URL. It's often used to test APIs or view the source of a webpage.

Remote Access & File Transfer

These tools are essential for interacting with other computers on a network.

  • FTP (File Transfer Protocol): A classic protocol for transferring files. Command-line clients like ftp and the more secure sftp (which uses SSH) are available.
  • ssh (Secure Shell): The primary tool for securely logging into another computer and running commands on it remotely. It provides an encrypted connection for all communication.
