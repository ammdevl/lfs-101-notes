Text Editor

Basic Editors: nano and gedit

Overview of Text Editors In Linux

Knowing how to use a text editor is an essential skill for Linux, required for tasks like editing system configuration files or writing scripts.

It's crucial to use a true text editor and not a word processor (like Microsoft Word or LibreOffice Writer). Word processors add hidden formatting that will break configuration files.

Linux offers many choices, including:

  • Simple/Beginner-friendly: nano and gedit.
  • Powerful/Advanced: vi and emacs.
  • Popular IDE: Visual Studio code is a more feature-rich option that many users are already familiar with.

Creating Files Without Using an Editor

For creating short files, you don't always need to open a full text editor. This is especially useful for quickly making a file from the command line or for creating files automatically inside a script.

One of the standard ways to do this is by using the echo command.

user@lfs101: ~
$ echo line one &gt; myfile
$ echo line two &gt;&gt; myfile
$ echo line three &gt;&gt; myfile
$ cat myfile
line one
line two
line three
$ cat &lt;&lt; EOF &gt; myfile
line one
line two
line three
EOF

Note that while a single greater-than sign (>) will send the output of a command to a file (and obliterate any already existing version of that file!), two of them (>>) will append the new output to an existing file.

nano and gedit

For beginners, Linux offers several easy-to-use text editors that don't require any special experience.

  • nano: A terminal-based editor that is great for new users because it always displays the basic commands at the bottom of the screen for easy reference. >_
  • gedit: A graphical editor for the GNOME desktop that is very intuitive and works a lot like the Notepad application from Windows. (The equivalent for the KDE desktop is kwrite or kate).

nano

nano is a beginner-friendly text editor that runs in the terminal. You can open or create a file by typing nano filename.

Its biggest advantage for new users is the shortcut bar at the bottom of the screen, which always shows the most important commands. The most essential ones to know are:

  • CTRL+O: Save the file (Write Out).
  • CTRL+X: Exit nano.
  • CTRL+G: Get help.
user@lfs101: ~
$ nano myfile

gedit

gedit is a graphical text editor for the Linux desktop. It's very easy to learn because its interface looks and feels a lot like Notepad from Windows.

However, gedit is much more powerful than Notepad. It is highly configurable and can be extended with plugins to add new features. You can start it from your desktop's application menu or by typing gedit filename in the terminal.

user@lfs101: ~
$ gedit myfile

Visual Studio Code

Visual Studio Code (code) is another simple-to-use graphical editor. It has a lot of functionality besides being used as a text editor.

More Advanced Editors: vi and emacs

vi and emacs

vi and emacs are the two classic, powerful text editors favored by experienced developers and system administrators.

They are famous for having a steep learning curve, making them difficult for beginners. However, once mastered, they are extremely efficient and powerful. Both can run in a basic terminal or as more user-friendly graphical applications.

There is a famous, long-standing rivalry (often called a "holy war") between users over which editor is superior, though vi is more widely used.

Introduction to vi

The text editor vi (pronounced "vee-eye") is usually a modern version called vim (Vi IMproved).

It is essential to learn the basics of vi because it is installed on virtually every Linux system, and sometimes it may be the only editor available.

vi is designed to be controlled entirely from the keyboard, which makes it very efficient once you learn the commands. For beginners, graphical versions like gvim are also available and may be easier to use.

user@lfs101: ~
$ vi myfile

vimtutor

The vimtutor command starts a short and effective built-in tutorial for learning vi.

By completing its seven lessons, you will learn a large number of commands and have all the skills needed to become a proficient vi user. It is the recommended first step for anyone who wants to learn the editor.

user@lfs101: ~
$ vimtutor

Modes in vi

vi provides three modes - command, insert and line.

ModeFeature
CommandBy default, vi starts in Command mode. Each key is an editor command. Keyboard strokes are interpreted as commands that can modify file contents.
InsertType i to switch to Insert mode from Command mode. Insert mode is used to enter (insert) text into a file. Insert mode is indicated by an “? INSERT ?” indicator at the bottom of the screen. Press Esc to exit Insert mode and return to Command mode.
LineType : to switch to the Line mode from Command mode. Each key is an external command, including operations such as writing the file contents to disk or exiting. Uses line editing commands inherited from older line editors. Most of these commands are actually no longer used. Some line editing commands are very powerful. Press Esc to exit Line mode and return to Command mode.

Using External Commands in vi

You can run shell commands directly from within vi in two main ways: >_

  1. :sh: This command opens a new, interactive command shell. You can run multiple commands, and when you type exit, you will return to your vi session right where you left off.
  1. :!: This is used to execute a single, non-interactive command. For example, the command :!wc % will run the wc (word count) program on the current file. In vi, the % symbol is a shortcut for the filename you are currently editing.

Introduction to emacs

emacs is a powerful, highly customizable text editor and the main competitor to vi.

The most significant difference from vi is that emacs does not use modes. You are always in "insert mode" and use key combinations with the CTRL and Meta (Alt) keys to execute commands.

emacs is known for being much more than just a text editor; it's an extensible environment that can handle tasks like email, debugging, and more.

Summary

Beginner-Friendly Editors

These editors are easy to learn and use, making them great for new users.

  • nano: A simple, text-based editor that runs in the terminal and always displays its commands on screen for easy reference.
  • gedit: A graphical editor, common on the GNOME desktop, that looks and feels very similar to Windows Notepad.

Powerful/Advanced Editors

These are the two classic, powerful editors favored by experienced developers and system administrators. They have a steep learning curve but are extremely efficient once mastered.

  • vi: Available on virtually every Linux system. It's famous for using different modes, such as a Command Mode for navigation and an Insert Mode for typing.
    • Tutorial: You can learn it by typing the command vimtutor.
  • emacs: A popular and highly extensible alternative to vi. It is modeless and instead uses key combinations with the Ctrl and Alt/Esc keys to perform commands.
    • Tutorial: You can access its tutorial by pressing Ctrl+h and then t from within the editor.
