String Manipulation

String Manipulation

A string is a sequence of text characters which can include letters, numbers, and symbols. Shell scripts provide operators to perform actions on strings, such as comparing them, sorting them, or finding their length.

When performing comparisons on strings in a bash script, it's a best practice to use the modern double square brackets ([[ ... ]]) syntax.

While the older single square brackets ([ ... ]) often work, the double bracket form is safer and helps avoid common errors, especially when working with variables that might be empty or contain spaces.

Parts of a String

You can extract a portion of a string (a substring) in bash using parameter expansion. Two common methods are by position and by pattern matching.

By Position and Length

This method lets you specify a starting point and the number of characters you want to extract. The syntax is: ${string:offset:length}

  • offset: The starting position (0 is the first character).
  • length: The number of characters to extract.

For example, ${MYVAR:0:4} would extract the first 4 characters from the variable $MYVAR.

By Removing a Prefix Pattern

This method removes a pattern from the beginning of a string. The syntax is: ${string#pattern}

This removes the shortest possible match of the pattern from the start of the string. For example, ${FILENAME#*.} is a common way to extract a file's extension by removing everything up to and including the first dot.

The case Statement

The case Statement

The case statement provides a clean way to choose a course of action from several different options, based on the value of a single variable. It compares the variable against a list of patterns and executes the block of code corresponding to the first match it finds.

Its main advantage is that it is much easier to read and less complex than using a long series of nested if-then-else statements. Because of this, it is commonly used for tasks like interpreting command-line options passed to a script.

Structure of The case Statement

Basic Structure

The case statement has a specific structure for testing the value of a variable against a list of possible patterns.

user@lfs101: ~
case $VARIABLE in
    pattern1)
        # Commands to run if $VARIABLE matches pattern1
        ;;
    pattern2)
        # Commands to run if $VARIABLE matches pattern2
        ;;
    *)
        # Default commands to run if no other pattern matches
        ;;
esac
  • case $VARIABLE in: Starts the statement, specifying which variable to test.
  • pattern): Each test is a pattern followed by a closing parenthesis.
  • ;; (double semicolon): Marks the end of the commands for a specific pattern. This is required.
  • ): A special wildcard pattern that acts as a default case, catching any value that doesn't match the other patterns.
  • esac: Ends the case statement (case spelled backwards).

Execution Flow

The case statement checks the patterns from top to bottom and stops at the first match. Once it finds a pattern that matches the variable's value, it executes the corresponding commands and then immediately exits the entire case block. No further patterns are tested.

Looping Constructs

Looping Constructs

What are Loops?

Loops are a fundamental programming construct that allows you to repetitively execute a block of code. They are essential for automating tasks that need to be performed multiple times, such as processing a list of files, until a specific condition is met.

The Three Main Loops in bash

Like many programming languages, bash provides three frequently used types of loops, each with a slightly different way of controlling the repetition.

  • for: This loop is typically used to iterate over a fixed list of items, executing a block of code once for each item in the list.
  • while: This loop will continue to execute a block of code as long as a specified condition remains true. It checks the condition before each iteration.
  • until: This loop is the opposite of a while loop. It will continue to execute a block of code until a specified condition becomes true. It also checks the condition before each iteration.

The for loop

How it Works

The for loop is a control structure designed to execute a block of code once for each item in a given list. It's like going through a checklist and performing the same action for every item on it until you reach the end.

Basic Syntax

The structure of a for loop is straightforward and easy to read.

user@lfs101: ~
for variable in item1 item2 item3
do
    # Commands to execute for each item
    # The current item is available in the &#x27;$variable&#x27;
done
  • for variable in list: This line initiates the loop. You define a temporary variable name that will hold each item from the list in turn.
  • do: Marks the beginning of the code block to be repeated.
  • done: Marks the end of the code block.

The while Loop

How it Works

The while loop is a control structure that executes a block of code repeatedly as long as a specified condition remains true. Before each iteration, the loop checks the condition. If it's true, the code runs. If it's false, the loop terminates.

Think of it like a traffic light: "while the light is green, keep going." As soon as the light turns red (the condition becomes false), you stop.

Basic Syntax

The structure of a while loop is simple and centers around the condition being tested.

user@lfs101: ~
while [ condition ]
do
    # Commands to execute as long as the condition is true
done
  • while [ condition ]: This line initiates the loop and defines the condition to be tested. The condition is often a test inside square brackets [].
  • do: Marks the beginning of the code block to be repeated.
  • done: Marks the end of the code block.

The until Loop

How it Works

The until loop is a control structure that executes a block of code repeatedly until a specified condition becomes true. It is the direct opposite of a while loop, which runs as long as a condition is true.

Think of it like a traffic light: "until the light is green, keep waiting." As soon as the light turns green (the condition becomes true), you stop. The condition is tested before each iteration.

Basic Syntax

The structure of an until loop is very similar to a while loop, but the logic is inverted.

user@lfs101: ~
until [ condition ]
do
    # Commands to execute until the condition is true
done
  • until [ condition ]: This line initiates the loop and defines the condition to be tested. The loop continues as long as this condition is false.
  • do: Marks the beginning of the code block to be repeated.
  • done: Marks the end of the code block.

Script Debugging

Debugging bash Scripts

When you work with scripts, you'll inevitably encounter errors. These can be caused by problems within the script itself, like syntax errors, or by external factors like a missing file or incorrect permission issues. Sometimes you'll get a specific error code, but often the only sign of a problem is confusing or incorrect output.

The process of identifying, troubleshooting, and fixing these problems is called debugging. It is an essential skill for anyone who writes scripts or manages a system.

Script Debug Mode

What is Debug Mode?

bash has a built-in debug mode that helps you find errors by tracing the execution of your script. When activated with the -x option, it prints each command to the terminal right before it's executed, prefixed with a + character. This allows you to see exactly what is running and with what values, making it easier to pinpoint the source of a problem.

How to Use Debug Mode

There are two primary ways to enable debug mode:

  • For the Entire Script: To debug a whole script from start to finish, run it by passing the x option to the bash interpreter: bash -x your_script.sh
  • For a Specific Section: To debug only a small part of a larger script, you can turn debugging on and off directly within the file. This is useful for isolating a problematic area.
user@lfs101: ~
# ... some code that works fine ...

set -x  # Turn debugging ON for this section
# ... the code you want to inspect ...
set +x  # Turn debugging OFF

# ... the rest of the script ...

Some Additional Useful Techniques

Creating Temporary Files and Directories

The Security Risk with Temporary Files

When scripts need to store data for a short time, they often use temporary files. However, creating these files with simple, predictable names (e.g., temp.txt) is insecure. An attacker could guess the filename and potentially read sensitive information from it or write malicious data into it before your script can.

The Secure Solution: mktemp

The correct and secure way to create temporary files is with the mktemp utility. It's designed specifically to generate a unique and random filename, which prevents these kinds of security vulnerabilities.

You use it by providing a filename template that ends in at least three X characters. mktemp will replace the Xs with random characters, creating and naming the secure temporary file for you.

For example: TMPFILE=$(mktemp /tmp/my_app.XXXXXX)

CommandUsage
TEMP=$(mktemp /tmp/tempfile.XXXXXXXX)To create a temporary file
TEMPDIR=$(mktemp -d /tmp/tempdir.XXXXXXXX)To create a temporary directory

Example of Creating a Temporary File and Directory

The Security Risk: Symbolic Link Attacks

Using predictable temporary files (e.g., /tmp/tempfile) in scripts is a major security vulnerability. It opens the door to a symbolic link attack.

Here's how it works:

  1. An attacker predicts the name of a temporary file that a script (especially one run by root) will use.
  1. Before the script runs, the attacker creates a symbolic link from that temporary filename to a critical system file, like /etc/passwd.
  1. When the script writes data to what it thinks is its temporary file, it's actually tricked into overwriting and corrupting the critical system file, which can cause severe damage.

The Safe Solution: mktemp

To prevent this type of attack, you should never use fixed, predictable names for temporary files. The correct and secure method is to use the mktemp utility.

mktemp generates a temporary file with a random, unpredictable name. This makes it impossible for an attacker to guess the filename in advance and create a malicious symbolic link.

Instead of

user@lfs101: ~
$ echo &quot;data&quot; &gt; /tmp/my_temp_file

You should do

user@lfs101: ~
$ TEMPFILE=$(mktemp /tmp/my_app.XXXXXX)echo &quot;data&quot; &gt; $TEMPFILE

Discarding Output with /dev/null

What is /dev/null?

The special file /dev/null is a core part of Linux, often called a "black hole" or "bit bucket." Its sole purpose is to accept any data written to it and immediately and permanently discard it. It's the perfect destination for command output that you want to ignore.

How to Use It

You can use standard redirection operators to send a command's unwanted output to /dev/null.

  • To discard normal output (stdout) only: Using command > /dev/null will send only the successful output to the black hole. Any error messages (stderr) will still be displayed on your screen.
  • To discard all output (stdout and stderr): Using the bash shortcut command >& /dev/null will send both the normal output and any error messages to /dev/null, making the command completely silent.

Random Numbers and Data

Generating random numbers is a common requirement for tasks like security, testing, and data erasure. Linux provides several ways to do this, each suited for different needs.

The Simple Method: $RANDOM

For basic scripting needs, the easiest way to get a random number is by using the built-in shell variable $RANDOM. Each time you reference it, the shell provides a pseudo-random integer between 0 and 32767. Its randomness is derived from the Linux kernel's own random number generator.

The Secure Method: OpenSSL

For high-security applications, especially encryption, you should not use $RANDOM. Instead, the OpenSSL library provides functions to generate cryptographically strong random numbers. These are based on certified algorithms (like FIPS 140) and are designed to be far more unpredictable and secure.

To learn about FIPS140, read Wikipedia's "FIPS 140-2" article.

How The Kernel Generates Random Numbers

The Entropy Pool

Linux generates random numbers by gathering unpredictable data, or "noise," from system events like mouse movements, keyboard timings, and network activity. This random data is collected and stored in what is known as an entropy pool. The kernel then provides two special device files that act as interfaces to draw random numbers from this pool.

/dev/random vs. /dev/urandom

While both devices provide random numbers, they have a critical difference in how they behave when the entropy pool runs low.

  • /dev/random: This device provides the highest quality of "true" randomness. However, it is a blocking device. If the entropy pool is empty, /dev/random will stop and wait until more environmental noise is collected. This makes it very slow but is preferred for extremely high-security tasks like generating long-term encryption keys.
  • /dev/urandom: This device is non-blocking. It provides fast, high-quality random numbers suitable for most applications. If the entropy pool runs low, it will reuse its existing internal state to continue generating pseudo-random numbers without waiting. Because of its speed and reliability, /dev/urandom is the recommended choice for almost all common cryptographic and general-purpose needs.

Summary

Advanced shell scripting involves a variety of techniques for making your scripts more powerful, robust, and intelligent. These include sophisticated ways to handle data, control the script's flow, and manage errors.

Conditional Logic & Flow Control

Making decisions is a core part of any script.

  • Boolean Expressions: These are tests that evaluate to either TRUE or FALSE. They can be used to compare numbers, strings, or check file properties. They are often combined using logical operators like && (AND), || (OR), and ! (NOT).
  • case Statement: An efficient control structure used to choose an action from a list of options based on the value of a single variable. It's often a cleaner alternative to complex, nested if statements.

Debugging & Output Handling

Finding and fixing errors is a critical skill.

  • Debugging: The process of troubleshooting and resolving errors in your scripts. bash provides tools, like debug mode (set -x), to help trace a script's execution.
  • Redirection: You can redirect a command's standard output (successful results) and standard error (error messages) into files. This is extremely useful for capturing logs and diagnosing problems.

Data Handling & Generation

Scripts often need to create and manipulate data on the fly.

  • String Manipulation: bash provides operators and utilities for performing actions on strings, such as comparing them, finding their length, or extracting substrings.
  • Temporary Files: You can create secure temporary files and directories to store data for a short period. Using tools like mktemp is essential for security.
  • Random Numbers: Linux offers several ways to generate random numbers, which are useful for a variety of tasks, from security to creating test data.
