Manipulating Text

cat and echo

Command Line Tools for Manipulating Text Files

Manipulating text files from the command line is an essential skill for any Linux user, whether you're an administrator, developer, or a regular user.

Tasks like searching for text and extracting data are done far more efficiently on the command line than with graphical tools. Furthermore, the command line makes it easy to automate these repetitive tasks by writing scripts.

cat

The cat command (short for concatenate) is a fundamental Linux utility with two main uses:

  1. Quickly displaying the contents of a file in your terminal.
  1. Its primary purpose: combining (concatenating) multiple files together.

The tac command (cat spelled backwards) does the opposite: it reads a file and prints its lines out in reverse order, from the last line to the first.

CommandUsage
cat file1 file2Concatenate multiple files and display the output; i.e. the entire content of the first file is followed by that of the second file
cat file1 file2 > newfileCombine multiple files and save the output into a new file
cat file >> existingfileAppend a file to the end of an existing file
cat > fileAny subsequent lines typed will go into the file, until CTRL-D is typed
cat >> fileAny subsequent lines are appended to the file, until CTRL-D is typed

Using cat Interactively

You can use the cat command to create files directly from the terminal without a text editor. This is done by redirecting the input from your keyboard into a file. There are two common ways to do this. 📄+

Using Redirection (>) and (>>)

This method reads every line you type until you signal the end of the file.

  1. Type cat > filename to create a new file (or overwrite an existing one). Use cat >> filename to append to an existing file.
  1. Enter the text you want in the file.
  1. On a new, empty line, press CTRL+D to save and exit.

Using a Here Document (<<)

A Here Document lets you specify a custom word to end the input, which is very useful in scripts.

  1. Type cat > filename << EOF. You can replace EOF with any word you like (e.g., STOP).
  1. Enter the text you want in the file.
  1. On a new, empty line, type the exact word you used after << (in this case, EOF) to save and exit.

echo

The echo command is a simple utility that displays a line of text to the terminal. It has several common uses:

  • Creating or appending text to files using the > and >> redirection operators.
  • Viewing the value of environment variables, such as echo $HOME.

Additionally, using the -e option allows you to format the output with special characters, like \n for a newline and \t for a tab.

CommandUsage
echo string > newfileThe specified string is placed in a new file
echo string >> existingfileThe specified string is appended to the end of an already existing file
echo $variableThe contents of the specified environment variable are displayed

Working with Large and Compressed Files

Working with Large Files

When you need to view a very large text file, like a log file, it's inefficient to use a standard text editor. Most editors try to load the entire file into memory, which can be very slow and resource-intensive.

The proper tool for this is the less command. less is a "pager" that lets you view the file one screen at a time without loading it all into memory, making it much faster. It's the same tool used by default to display man pages.

It's an improvement on an older, similar command called more, leading to the common Linux joke: less is more!

user@lfs101: ~
$ less myfile.txt

head

head reads the first few lines of each named file (10 by default) and displays it on standard output. You can give a different number of lines in an option

user@lfs101: ~
$ head -3 myfile.txt
line one
line two
line three

tail

tail prints the last few lines of each named file and displays it on standard output. By default, it displays the last 10 lines. You can givea different number of lines as an option. tail is especially useful when you are troubleshooting any issue using log files, as you probably want to see the most recent lines of output.

user@lfs101: ~
$ tail -3 myfile.txt
line eight
line nine
line ten

Viewing Compressed Files

Standard commands like cat and less can't read compressed files directly.

To work with these files without decompressing them first, you can use special "z" versions of common utilities. These commands often have a "z" prefixed to their name.

For example, you can use zcat instead of cat, zless instead of less, zdiff instead of diff, and zgrep instead of grep.

CommandDescription
$ zcat compressed-file.txt.gzTo view a compressed file
$ zless somefile.gz or$ zmore somefile.gzTo page through a compressed file
$ zgrep -i less somefile.gzTo search inside a compressed file
$ zdiff file1.txt.gz file2.txt.gzTo compare two compressed files

Commands like zless are flexible; they work correctly even on uncompressed files, simply skipping the decompression step.

It's important to know which prefix to use, as it corresponds to the file's compression method:

  • The z prefix (e.g., zcat, zless) is for files compressed with gzip.
  • The xz prefix (e.g., xzcat, xzless) is for files compressed with xz.
  • The bz prefix (e.g., bzcat, bzless) is for files compressed with bzip2.

sed and awk

Introduction to sed and awk

The sed and awk utilities are classic command-line tools for editing files and extracting data.

While many people now use full scripting languages like Python or Perl for these jobs, sed and awk have important advantages:

  • They are much lighter, using fewer system resources.
  • They execute faster.

Because of this efficiency, they are essential in constrained environments, like during the system's boot process, where a heavier language might be too slow or unavailable.

sed

The sed command, short for stream editor, is a powerful tool for modifying text from a file or an input stream. Its most common uses are filtering text and performing substitutions (find and replace).

It works by reading the input line by line, applying a set of commands to each line in a temporary workspace, and then printing the modified result. By default, sed does not change the original file; it only outputs the modified version to the screen or another file.

sed Command Syntax

You can invoke sed using commands like those listed in the accompanying table.

CommandUsage
sed -e command <filename>Specify editing commands at the command line, process input from a file, and put the output on standard out (e.g., the terminal)
sed -f scriptfile <filename>Specify a script file containing sed commands, operate on file, and put output on standard out
echo "I hate you" | sed s/hate/love/Use sed to filter standard input, putting output on standard out

The -e option allows you to specify multiple editing commands simultaneously at the command line. It is unnecessary if you only have one operation invoked.

sed Basic Operations

A very common use of sed is for substitution (find and replace) using the format 's/pattern/replace_string/g'.

While sed has an -i option to edit files directly ("in-place"), this is dangerous because the changes are permanent and cannot be undone.

The recommended safe workflow is:

  1. Run the sed command and redirect the output to a new temporary file: sed 's/old/new/g' original_file > temp_file
  1. Check the temp_file to verify the changes are correct.
  1. If they are, overwrite the original file with the temporary one: mv temp_file original_file
CommandUsage
sed s/pattern/replace_string/ fileSubstitute first string occurrence in every line
sed s/pattern/replace_string/g fileSubstitute all string occurrences in every line
sed 1,3s/pattern/replace_string/g fileSubstitute all string occurrences in a range of lines
sed -i s/pattern/replace_string/g fileSave changes for string substitution in the same file

$ cat myfile.txt

line two
line three
line four
line five
line six
line seven
line eight
line nine
line ten
$ sed -e &#x27;s/one/onee/g&#x27; myfile.txt &gt; temp.txt
$ cat temp.txt

line two
line three
line four
line five
line six
line seven
line eight
line nine
line ten

awk

The awk utility is a powerful tool designed to extract data from a file and print it in a formatted way, making it excellent for creating reports. Its name comes from the initials of its creators: Aho, Weinberger, and Kernighan.

awk's main strength is its ability to process text as a collection of records (which are typically lines) that are split into fields (which are essentially columns). This makes it ideal for manipulating structured data. Beyond being a simple command, awk is also a complete programming language.

CommandUsage
awk ‘command’ fileSpecify a command directly at the command line
awk -f scriptfile fileSpecify a file that contains the script to be executed

awk Operations

The most basic function of awk is to print specific columns (or "fields"). By default, awk uses whitespace to separate fields.

  • $0 refers to the entire line.
  • $1 refers to the first field, $2 to the second, and so on.
CommandUsage
awk '{ print $0 }' /etc/passwdPrint entire file
awk -F: '{ print $1 }' /etc/passwdPrint first field (column) of every line, separated by a colon
awk -F: '{ print $1 $7 }' /etc/passwdPrint first and seventh field of every line
user@lfs101: ~
$ cat &gt; myfile.txt
ID Name Dept Salary
101 Alice Sales 70000
102 Bob   Eng   85000
103 Carol Sales 92000
104 David Eng   78000
$ awk &#x27;{print $0}&#x27; myfile.txt
ID Name Dept Salary
101 Alice Sales 70000
102 Bob   Eng   85000
103 Carol Sales 92000
104 David Eng   78000
$ awk &#x27;{print $2, $3}&#x27; myfile.txt
ID Name Dept Salary
Alice Sales
Bob   Eng
Carol Sales
David Eng

File Manipulation Utilities

Overview

Linux offers a powerful toolbox of command-line utilities for manipulating text files, such as sorting and organizing data.

This section introduces several of these key programs:

  • sort
  • uniq
  • paste
  • join
  • split

It will also cover the use of regular expressions for advanced text searching and matching.

sort

The sort command is used to rearrange the lines in a text file. By default, it sorts the lines alphabetically. You can also configure it to sort based on a specific field (column).

A very common option is -u (unique), which tells sort to remove any duplicate lines from its output. Using sort -u is a convenient shortcut for the command pipeline sort | uniq.

SyntaxUsage
sort <filename>Sort the lines in the specified file, according to the characters at the beginning of each line
cat file1 file2 | sortCombine the two files, then sort the lines and display the output on the terminal
sort -r <filename>Sort the lines in reverse order
sort -k 3 <filename> Sort the lines by the 3rd field on each line instead of the beginning
user@lfs101: ~
$ cat myfile.txt
line one 
line two
line three
line four
line five
$ sort myfile.txt
line five
line four
line one
line three
line two

uniq

The uniq command is used to remove duplicate lines from a text file.

The most important rule to remember is that it only removes duplicate lines that are consecutive (right next to each other). Because of this, you almost always need to sort the file first. The standard command pipeline is: sort file | uniq

(Remember that sort -u does the same job more efficiently.)

Additionally, using the -c (count) option with uniq will count the number of occurrences of each line instead of removing the duplicates.

paste

The paste command merges lines from multiple files together side-by-side (in parallel), like a zipper combining two halves. It's perfect for when you have columns of related data in separate files and want to combine them into a single file.

By default, the columns in the new, combined line are separated by a tab.

Key options include:

  • d (delimiter): Lets you specify a different separator, like a comma.
  • s (series): Changes the behavior to combine all lines from one file into a single horizontal line, rather than merging files in parallel.
user@lfs101: ~
$ cat myfile1.txt
E001 23078
E002 23075
E003 23045
E004 23098
E005 23094
$ cat myfile2.txt
Mg Mg          
Kyaw Kyaw
Ko Ko
Nyi Nyi
Min Min
$ paste myfile1.txt myfile2.txt
E001 23078 Mg Mg
E002 23075 Kyaw Kyaw
E003 23045 Ko Ko
E004 23098 Nyi Nyi
E005 23094 Min Min

join

The join command is used to merge the lines of two files that share a common field (or column).

Think of it like a database join: it looks for matching data in the specified column and combines the corresponding lines from each file into a single line. This makes it more intelligent than paste, as it doesn't just place files side-by-side but actively matches up related data.

user@lfs101: ~
$ cat myfile1.txt
E001 23078
E002 23075
E003 23045
E004 23098
E005 23094
$ cat myfile2.txt
E001 Mg Mg          
E002 Kyaw Kyaw
E003 Ko Ko
E004 Nyi Nyi
E005 Min Min
$ join myfile1.txt myfile2.txt
E001 23078 Mg Mg
E002 23075 Kyaw Kyaw
E003 23045 Ko Ko
E004 23098 Nyi Nyi
E005 23094 Min Min

split

The split command is used to break a large file into multiple smaller pieces, making it easier to view and manage.

By default, it splits the file into segments of 1000 lines each. The original file remains unchanged, and a new set of smaller files is created. By default, these new files are named with the prefix x (e.g., xaa, xab), but you can also specify your own custom prefix.

We use wc (word count) to report on the number of lines in the file.

user@lfs101: ~
$ wc -l linux.words
99171 american-english
$ split linux.words lwords

Now,the American-English file splits into equal-sized segments named lwordsxx.

Regular Expressions and Search Patterns

A regular expression (often called regex) is a special string of text used for advanced pattern matching. It uses a mix of normal characters and special meta-characters (like *, $, ^) to define a search pattern.

It's important to understand that regex is different from and much more powerful than the simple wildcards (*, ?) used for filename matching in the shell. Many powerful command-line tools like grep, sed, and awk rely on regular expressions to find and manipulate text.

Search PatternsUsage
.(dot)Match any single character
a|zMatch a or z
$Match end of a line
^Match beginning of a line
*Match preceding item 0 or more times

For example, consider the following sentence: the quick brown fox jumped over the lazy dog.

Some of the patterns that can be applied to this sentence are as follows:

CommandUsage
a..matches azy
b.|j.matches both br and ju
..$matches og
l.*matches lazy dog
l.*ymatches lazy
the.*matches the whole sentence

grep and strings

grep

The grep command is a fundamental and widely used tool for text searching in Linux. It scans files for lines that match a specified pattern. You can use it to search for simple text strings or for complex patterns by using regular expressions.

CommandUsage
grep [pattern] <filename>Search for a pattern in a file and print all matching lines
grep -v [pattern] <filename>Print all lines that do not match the pattern
grep [0-9] <filename>Print the lines that contain the numbers 0 through 9
grep -C 3 [pattern] <filename>Print context of lines (specified number of lines above and below the pattern) for matching the pattern. Here, the number of lines is specified as 3
user@lfs101: ~
$ cat myfile.txt
The quick brown fox jumps over the lazy dog.
A slow brown cat naps near the doghouse.
What will the fox do next?
The lazy dog is very lazy.
$ grep &quot;fox&quot; myfile.txt 
The quick brown fox jumps over the lazy dog.
What will the fox do next?
OptionMeaning
-iIgnore case
-vInvert match
-nPrint line number
-HPrint filename
-aTreat binary files as text
-IIgnore binary files
-rRecurse through subdirectories
-lPrint out names of all files that contain matches
-LPrint out names of all files that do not contain matches
-cPrint out number of matching lines only
-eUse the following pattern; useful for multiple strings and special characters

strings

The strings command is used to extract all human-readable text from a binary file, like an executable program or a spreadsheet.

While you'd use grep to search a normal text file, strings is the tool for finding text embedded within non-text files. A common and powerful technique is to pipe the output of strings into grep to first extract all the text and then search for a specific pattern within it. For example: strings program_file | grep "some_text"

user@lfs101: ~
$ cat myfile.txt
The quick brown fox jumps over the lazy dog.
A slow brown cat naps near the doghouse.
What will the fox do next?
The lazy dog is very lazy.
$ strings myfile.txt | grep &quot;dog&quot;
The quick brown fox jumps over the lazy dog.
A slow brown cat naps near the doghouse.
The lazy dog is very lazy.

Miscellaneous Text Utilities

tr

The tr command is a simple utility used to translate (substitute) or delete individual characters from a stream of text.

It reads from standard input, finds every character specified in a first set (set1), and replaces it with the corresponding character from a second set (set2).

A classic example is converting a file's text from lowercase to uppercase: cat filename | tr 'a-z' 'A-Z'

It's a good practice to enclose the character sets in single quotes (' ') to prevent the shell from misinterpreting them.

user@lfs101: ~
$ cat allcaps.txt
LINE ONE
LINE TWO
LINE THREE
LINE FOUR
LINE FIVE
$ cat allcaps.txt | tr &#x27;A-Z&#x27; &#x27;a-z&#x27;
line one
line two
line three
line four
line five
CommandUsage
tr abcdefghijklmnopqrstuvwxyz ABCDEFGHIJKLMNOPQRSTUVWXYZConvert lower case to upper case
tr '{}' '()' < inputfile > outputfileTranslate braces into parenthesis
echo "This is for testing" | tr [:space:] '\t'Translate white-space to tabs
echo "This is for testing" | tr -s [:space:]Squeeze repetition of characters using -s
echo "the geek stuff" | tr -d 't'Delete specified characters using -d option
echo "my username is 432234" | tr -cd [:digit:]Complement the sets using -c option
tr -cd [:print:] < file.txtRemove all non-printable character from a file
tr -s '\n' ' ' < file.txtJoin all the lines in a file into a single line

tee

The tee command is named after a T-shaped pipe fitting, and it works the same way: it's used to split the output of a command into two separate streams.

It takes an input (usually from a pipe |) and simultaneously:

  1. Displays the output on the screen (standard output).
  1. Saves the exact same output to a file.

For example, the command ls -l | tee file.txt will show the directory listing on your screen and save it to file.txt at the same time.

wc

wc (word count) counts the number of lines, words, and characters in a file or list of files.

OptionDescription
-lDisplays the number of lines
-cDisplays the number of bytes
-wDisplays the number of words

cut

The cut command is used to extract specific columns of text from a file or data stream.

By default, it assumes the columns are separated by a TAB character. You can easily select which columns to keep and what character separates them using these key options:

  • d (delimiter): Specifies the character that separates the columns (e.g., d" " for a space).
  • f (field): Specifies the column number(s) you want to extract.

For example, ls -l | cut -d" " -f3 would extract the third column from the ls -l output, using a space as the delimiter.

Summary

Viewing & Displaying Files

  • cat: Reads, prints, and combines (concatenates) files.
  • echo: Displays a specified line of text.
  • less: Views files one page at a time, allowing scrolling.
  • head: Displays the first few lines of a file (10 by default).
  • tail: Displays the last few lines of a file (10 by default).

Searching & Pattern Matching

  • Regular Expressions: Special text strings used for advanced pattern matching.
  • grep: Searches for patterns within text files and data streams.
  • strings: Extracts human-readable text from binary (non-text) files.

Editing & Transforming Data

  • sed: A stream editor used to filter and perform find-and-replace on text.
  • awk: A programming language for advanced data extraction and reporting.
  • tr: Translates or deletes individual characters.
  • cut: Extracts specific columns of text.

Organizing & Combining Files

  • sort: Rearranges the lines of a text file in alphabetical or numerical order.
  • uniq: Removes duplicate consecutive lines from a file.
  • paste: Merges lines from multiple files together side-by-side.
  • join: Combines lines from two files based on a matching common field.
  • split: Breaks a single large file into smaller pieces.

Other Essential Utilities

  • tee: Splits the output of a command, displaying it on the screen while also saving it to a file.
  • wc: Counts the number of lines, words, and characters in a file.
  • z command family: A set of utilities (like zcat, zless) for working directly with compressed files.
