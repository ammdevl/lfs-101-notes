Finding Linux Documentation

Linux Documentation Sources

Whether you're a novice or an expert user, you'll occasionally struggle to recall the proper use of Linux programs and utilities—such as which commands to use, what options to select, or what results to expect. You'll likely need to consult help documentation regularly. Linux-based systems draw from diverse sources, creating numerous documentation repositories and help methods. Distributors perform the valuable service of consolidating this material and presenting it in a comprehensive, user-friendly manner—an essential function of any Linux distribution.

Key Linux documentation resources include:

  • The man pages (short for manual pages)
  • GNU Info documentation system
  • The help command and --help option
  • Distribution-specific documentation, including Gentoo Handbook, Ubuntu Documentation, and Fedora Documentation.

The man pages

The man pages are the primary Linux documentation source, providing comprehensive information about programs, utilities, configuration files, and programming APIs. They are available on all Linux distributions.

Access information by typing "man" followed by a topic name.

Dating back to early UNIX in the 1970s, man simply stands for manual.

Man pages are available in various formats including PDF and web pages. Visit Linux man pages online to learn more. Many sites offer graphical interfaces for help content.

Additional documentation sources include published books and various Internet resources.

man

The man program searches, formats, and displays information from the man page system. Since topics often contain extensive information, output is piped through a pager program (like less) for page-by-page viewing with proper formatting.

Topics may have multiple pages with a default display order when no options or section is specified. Use -f to list all pages on a topic, or -k to list all pages discussing a specific subject (even when not in the name).

  • man –f generates the same result as typing whatis.
user@lfs101: ~
$ man -f sysctl | whatis sysctl
sysctl (2)           - read/write system parameters
sysctl (8)           - configure kernel parameters at runtime
  • man –k generates the same result as typing apropos.
user@lfs101: ~
$ man -k sysctl | apropos
_sysctl (2)          - read/write system parameters
proc (5)             - process information, system information, and sysctl ps...
proc_sys (5)         - system information, and sysctl pseudo-filesystem
procfs (5)           - process information, system information, and sysctl ps...
sysctl (2)           - read/write system parameters
sysctl (8)           - configure kernel parameters at runtime
sysctl.conf (5)      - sysctl preload/configuration file
sysctl.d (5)         - Configure kernel parameters at boot
systemd-sysctl (8)   - Configure kernel parameters at boot
systemd-sysctl.service (8) - Configure kernel parameters at boot

Manual Chapters

Man pages are organized into chapters 1-9. Letters may be added to chapter numbers for specific topics, like 3X for X Window API pages.

You can specify a chapter number to view a particular man page when multiple pages share the same name across chapters, which is common for library functions and system calls.

Use the -a parameter to display all pages with the same name across all chapters sequentially:

user@lfs101: ~
$ man -a socket

GNU Info

The GNU Info System

The GNU Info System is another important source of Linux documentation.

This is the GNU project's standard documentation format, which they prefer as an alternative to man. The Info System is free-form and features linked subsections.

Functionally, info resembles man in many ways. However, it organizes topics using links (despite being designed before the World Wide Web). You can access information through a command line interface, a graphical help utility, or view it online or in print.

Although the info interface may appear outdated compared to modern help systems (even man), it often provides more comprehensive information. For example, comparing the output of a simple command reveals much more detail in the info page (try man ls vs info ls and note the difference in content). This makes learning how to use info still valuable.

user@lfs101: ~
$ man ls
user@lfs101: ~
$ info ls

Using info in The Command Line

Running info without arguments displays an index of available topics. Navigate using arrow keys, Page Up, and Page Down.

For specific help, type info <topic name> to search across all info files.

Key commands: q (quit), h (help), Enter (select menu item).

user@lfs101: ~
$ info

The --help Option and help Command

The --help Option

The --help option is another key Linux documentation source.

Most commands offer a brief description when you use --help or -h. For example, to get information about the man command, type:

user@lfs101: ~
$ man --help

The help Command

Within a bash command shell, some popular commands (such as echo and cd) actually execute built-in bash versions rather than the standard binaries found in /bin or /usr/bin directories. This approach is more efficient and faster because it uses fewer system resources (we will discuss command shells later). Note that there may be minor differences between the built-in and standard versions of these commands.

user@lfs101: ~
$ help

Other Documentation Sources

In addition to the man pages, the GNU Info System, and the help command, Linux offers several other documentation sources:

  • Desktop help system
  • Package documentation
  • Online resources

Graphical Help Systems

Linux desktop systems feature graphical help applications, typically shown as question-mark icons or life preservers and accessible through the menu system. These include desktop-specific help, application guidance, and often graphically-formatted info and man pages.

Instead of searching for icons or menu items, you can launch the graphical help system directly from a terminal window or command prompt using one of these utility programs:

GNOME

user@lfs101: ~
$ gnome-help

KDE

user@lfs101: ~
$ khelpcenter

Package Documentation

Documentation is available through the package management system, with content pulled from upstream source code and information about distribution packaging.

This is stored in the /usr/share/doc directory, organized by package name.

Online Resources

Online Linux documentation is abundant and easily accessible.

For a comprehensive resource, check out "The Linux Command Line" by William Shotts - a free command line guide under Creative Commons license.

Each distribution provides its own documentation through user forums and wikis:

  • Ubuntu Documentation
  • CentOS Documentation
  • openSUSE Documentation
  • Gentoo Documentation
  • Fedora Documentation

Search engines can also help you find resources like blog posts, forum discussions, and articles.

Summary

  • man pages: The traditional and most common source for in-depth manuals. Accessed with the man command (e.g., man ls), they provide comprehensive documentation for commands, system calls, and configuration files.
  • info pages: The official documentation system from the GNU project, often providing more structured, hyperlink-style information than man pages. They are accessed with the info command.
  • command **--help*: A standard option you can pass to most commands (e.g., ls --help) to get a quick usage summary and a list of available options directly in your terminal.
  • help command: A command used to get help specifically for built-in shell commands (like cd or echo).
